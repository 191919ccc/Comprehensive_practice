# 随机森林完整训练进度显示修改说明

## 修改目的

正式日线训练在随机森林阶段耗时很长，原日志只停留在 `[ml] training random_forest direction_threshold=0.02`，无法判断当前训练进展。同时用户希望完整重新训练一次，不跳过随机森林校准和 walk-forward 回测。本次修改恢复完整训练链路，并增加长耗时阶段的进度显示。

## 修改文件

- `python/ml/stock_ml.py`
- `python/common/config.py`

## 具体修改

1. 增加训练进度心跳
   - 新增 `run_with_progress()`。
   - 阶段开始打印 `start`。
   - 阶段运行中每 30 秒打印一次 `running, elapsed=...s`。
   - 阶段结束打印 `done, cost=...s`。

2. 随机森林默认完整训练
   - `ML_RF_FULL_CALIBRATION` 默认值改为 `true`，默认启用 `CalibratedClassifierCV(cv=3)`。
   - `ML_RF_WALK_FORWARD` 默认值改为 `true`，默认执行随机森林 walk-forward 回测。
   - 不再默认跳过任何随机森林评估阶段。

3. 随机森林并行度可调
   - 新增 `ML_RF_N_JOBS`，默认值为 `4`。
   - 随机森林回归模型和分类模型都会使用该并行度。
   - 训练开始会打印当前 `random_forest n_jobs=...`。

## 完整重新训练命令

建议先在当前训练终端按 `Ctrl+C` 中断旧训练，再重新执行：

```powershell
$env:ML_RF_FULL_CALIBRATION="true"
$env:ML_RF_WALK_FORWARD="true"
$env:ML_RF_N_JOBS="4"
D:\anaconda3\envs\MachineLearn\python.exe -m python.ml.train_daily_predict --version-label daily-best
```

如果 CPU 仍然偏低，可以把 `ML_RF_N_JOBS` 调高，例如：

```powershell
$env:ML_RF_N_JOBS="8"
```

如果 Windows 再出现 joblib 权限或子进程错误，则把它降回：

```powershell
$env:ML_RF_N_JOBS="1"
```

## 剩余限制

- 完整校准和 walk-forward 都开启后，随机森林训练仍可能耗时较长。
- `ML_RF_N_JOBS` 提高后 CPU 利用率会上升，但 Windows 环境可能更容易触发 joblib 子进程权限问题。
