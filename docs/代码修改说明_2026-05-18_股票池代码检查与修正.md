# 股票池代码检查与修正说明

日期：2026-05-18

## 检查目标

检查实时采集股票池中是否存在错误股票代码、市场归属不匹配、重复代码，以及会导致采集失败的配置问题。

## 检查范围

- `python/data/stock_symbols.json`
- `python/common/stock_utils.py`
- `python/producer/stock_sources.py`
- MySQL 表：`price_ticks`、`daily_stock_bars`

## 检查结果

### 1. 股票池配置文件

当前股票池配置检查后为 101 只：

- SH：46
- SZ：45
- HK：10

检查项全部通过：

- 无重复代码。
- 无非法市场类型。
- A 股代码均为 6 位数字。
- 港股代码均为 5 位数字。
- `market` 字段和代码前缀匹配。
- 新浪、腾讯、东方财富代码转换函数均可生成采集代码。

### 2. 发现并修正的问题

#### 删除 `601989`

`601989` 不再适合作为实时股票池采集代码，已从 `python/data/stock_symbols.json` 删除。

#### 补回 `000338`

数据库 `daily_stock_bars` 和前端名称映射中已有 `000338`，但实时股票池缺少该代码。已将 `000338` 补回实时股票池：

```json
{"symbol":"000338","company_name":"Weichai Power","category":"A Manufacturing","sector":"Auto Parts","market":"SZ"}
```

### 3. 发现的历史数据残留

`price_ticks` 表中仍有旧的美股历史数据：

```text
AAPL, AMD, AMZN, BAC, CRM, CVX, GOOGL, INTC, JPM, KO, MCD, META, MSFT, NVDA, ORCL, PEP, TSLA, XOM
```

这些代码已经不在当前 `python/data/stock_symbols.json` 股票池中，属于历史入库残留，不是当前实时采集股票池错误。

## 采集校验修正

检查时发现部分代码不是代码错误，而是采集校验逻辑会误判：

1. 港股使用新浪接口时，部分成交量字段会返回 0 或负数，导致被 `is_valid_tick()` 过滤。
2. 创业板 `300/301` 和科创板 `688` 仍按 10% 涨跌幅限制过滤，会误杀正常 20% 涨跌幅范围内的行情。

已修改：

- `python/common/stock_utils.py`
  - 创业板、科创板涨跌幅过滤放宽到 20% 范围。
  - 北交所代码预留 30% 范围。
- `python/producer/stock_sources.py`
  - 单个数据源返回无效行情时，不再直接返回。
  - 继续尝试下一个数据源，例如港股新浪无效时自动切到腾讯。
