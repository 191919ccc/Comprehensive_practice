# 模型验证前端展示口径修正说明

## 修改目的

首页“模型验证状态”卡片原来把 `balanced_direction_accuracy` 和 `majority_baseline_accuracy` 放在一起比较，导致 LightGBM 平衡准确率 41.0% 被错误显示为“低于基线 46%”。这两个指标口径不同，不能直接比较。

## 修改文件

- `frontend/index.html`
- `frontend/dash-pages.js`

## 具体修改

1. 新增前端 `modelBenchmark()` 判断展示基准。
2. `balanced_direction_accuracy` 改为对比随机三分类基线 `33.3%`。
3. `direction_macro_f1` 改为对比随机宏 F1 参考 `33.3%`。
4. `direction_accuracy` 继续对比 `majority_baseline_accuracy`。
5. 首页文案从“低于基线，谨慎参考”改为按真实比较口径显示“高于随机三分类基线”或“低于对应参考”。
6. 模型分析页的说明文案同步使用同一套比较口径。

## 验证命令

```powershell
node --check frontend\dash-pages.js
```

## 验证结果

两个前端脚本语法检查均通过。当前 `http://localhost:5500/index.html` 已能读取到新的 `modelBenchmark()` 逻辑。

## 剩余说明

这次修正的是展示口径，不是把模型指标“美化”。如果看 `direction_accuracy`，仍应和多数类基线比较；如果看 `balanced_direction_accuracy`，更合理的参考是三分类随机水平 `33.3%`。
