# 代码修改说明：机器学习模型训练与漂移评估改进

## 1. 修改时间

2026-05-09

## 2. 修改目标

本次修改围绕机器学习模型效果偏低、`balanced_accuracy` 接近随机水平、模型漂移检测口径不一致、LSTM 序列抽样破坏训练稳定性等问题展开。

修改目标不是简单调参，而是让训练指标更可信、漂移检测更接近训练标签定义、LSTM 保留更合理的连续时间窗口，并在训练结果中输出更细的诊断指标，方便判断模型到底错在哪里。

## 4. 具体修改内容

### 4.1 增加方向分类诊断指标

修改位置：

```text
python/ml/stock_ml.py -> direction_eval_metrics()
```

原来训练结果主要输出：

- `direction_accuracy`
- `balanced_direction_accuracy`
- `direction_macro_f1`
- `majority_baseline_accuracy`
- `validation_down_ratio`
- `validation_up_ratio`
- `validation_flat_ratio`

本次新增：

- `down_recall`
- `down_precision`
- `predicted_down_ratio`
- `up_recall`
- `up_precision`
- `predicted_up_ratio`
- `watch_recall`
- `watch_precision`
- `predicted_watch_ratio`

这样可以更清楚判断模型问题。例如：

- 如果 `predicted_watch_ratio` 很高，说明模型过度保守。
- 如果 `up_recall` 很低，说明上涨样本识别差。
- 如果 `down_precision` 很低，说明看空信号误报多。
- 如果 `balanced_accuracy` 接近 0.33，说明三分类几乎接近随机水平。

### 4.2 修正 LSTM 序列抽样方式

修改位置：

```text
python/ml/stock_ml.py -> build_lstm_samples()
```

原逻辑：

```python
sample_index = np.linspace(0, len(x_array) - 1, settings.ml_lstm_max_sequences, dtype=int)
```

问题：

- `np.linspace` 会在全局时间线上均匀抽取 LSTM 样本。
- 虽然单个窗口内部仍是连续的，但会稀疏化每只股票保留下来的训练窗口。
- 对 LSTM 来说，最近一段连续市场状态通常比全局稀疏样本更有价值。

新逻辑：

- 按股票分组。
- 每只股票保留最新的一段 LSTM 窗口。
- 总样本数仍受 `ML_LSTM_MAX_SEQUENCES` 控制。
- 保留样本时尽量保持每只股票的最新连续训练窗口。

训练时现在会打印类似信息：

```text
[ml] limit LSTM sequences 15287 -> 6000 with per-symbol latest windows
```

### 4.3 统一 model_drift 实际方向阈值

修改位置：

```text
python/ml/model_drift.py -> actual_signal()
```

原逻辑：

```python
if actual_price > current_price * (1 + 1e-6):
    return "UP"
if actual_price < current_price * (1 - 1e-6):
    return "DOWN"
return "WATCH"
```

问题：

- 训练标签使用的最低方向阈值是 `0.005`，也就是 0.5%。
- drift 检测使用 `1e-6`，几乎任何微小价格变化都会被判成 `UP` 或 `DOWN`。
- 这会导致模型预测 `WATCH` 时大量被判错，漂移准确率失真。

新逻辑：

```python
DEFAULT_SIGNAL_THRESHOLD = 0.005
```

现在 drift 检测默认也使用 0.5% 作为实际方向阈值，小于该阈值的真实变化会归为 `WATCH`。同时命令行新增参数：

```powershell
--signal-threshold
```

用于后续手动调整 drift 检测口径。

## 6. 本次修改后的结论

本次修改验证了之前的判断：

- `model_drift.py` 的阈值不一致确实会导致 drift 评估失真。
- `np.linspace` 抽样会影响 LSTM 的训练稳定性。
- 修正后 LSTM 的 balanced accuracy 有明显改善，达到约 0.4016。
- 但整体模型仍未达到稳定可用水平，根本瓶颈仍然是数据量、数据质量、特征信息和标签难度。

## 7. 后续建议

下一步最值得继续做的是：

1. 增加更多股票和更长周期的日线数据。
2. 增加大盘指数、行业指数、市场涨跌家数等外部市场特征。
3. 对 `UP / DOWN / WATCH` 阈值做实验，例如 0.3%、0.5%、1.0% 对比。
4. 增加 walk-forward 回测，不只做一次 8:2 时间切分。
5. 把 drift 检测从“预测后一条真实行情”改成“预测 horizon 后的真实行情”，与 `PREDICTION_HORIZON=3` 完全一致。
