# 代码修改说明：风险告警依据与股票历史导出

## 修改目的

本次修改是为了解决风险告警页展示可信度不足的问题。原页面只展示告警列表和基础字段，无法说明每条告警为什么触发；同时股票历史和导出功能已有前端逻辑雏形，但独立风险告警页没有入口，用户容易以为功能缺失。

## 修改文件

- `frontend/alerts.html`
- `frontend/dash-pages.js`
- `frontend/dash-pages.css`
- `frontend/index.html`
- `docs/代码修改说明_2026-05-27_风险告警依据与股票历史导出.md`

## 具体修改

1. 在 `index.html` 首页大屏主图工具栏增加“历史记录”和“导出CSV”，位置与“实时 / 日线+预测”同属价格走势可视化区域。
2. 在 `dash-pages.js` 中恢复 `alertTabs` 点击监听，使“全部 / 价格异动 / 成交量 / 模型告警”筛选可用。
3. 告警等级优先使用后端返回的 `alert_level`，前端只在字段缺失时兜底推断，避免页面显示与数据库记录不一致。
4. 每条告警增加“查看依据”按钮，点击后懒加载该股票最近 30 分钟行情，并展示：
   - 当前涨跌幅与价格告警阈值对比；
   - 当前成交量与成交量告警阈值对比；
   - 最近价格走势小图和触发点；
   - 模型类告警的来源说明。
5. 阈值估算沿用 Spark 告警规则：
   - 价格告警阈值：`max(2.0, 历史平均绝对涨跌幅 * 2.5)`；
   - 高危价格阈值：`max(4.0, 历史平均绝对涨跌幅 * 4.0)`；
   - 成交量告警阈值：`max(市场最低量, 历史平均成交量 * 2.0)`；
   - 高危成交量阈值：`max(市场最低量 * 2.0, 历史平均成交量 * 4.0)`。
6. “历史记录”调用现有 `/api/history` 接口，在首页主图下方展开股票历史表格；“导出CSV”调用现有 `/api/history/export` 接口，导出当前首页图表选中股票的数据。
7. 告警页不再放置股票历史/导出入口，避免把股票行情历史误解为告警记录导出。
8. 告警处理按钮增加处理中状态和失败提示，避免网络慢或后端异常时只在控制台报错。
9. 首页告警卡片同步改为优先使用后端 `alert_level`，并给处理按钮增加失败反馈；首页不增加复杂展开图表。

## 解决的问题

- 风险告警不再只是列表展示，单条告警可以展开看到触发证据。
- 告警页的分类筛选按钮可以正常工作。
- 股票历史和 CSV 导出入口在风险告警页可见。
- 前端展示的告警等级与后端数据库记录保持一致。
- 告警状态更新失败时用户能看到提示。

## 验证命令

```powershell
node --check frontend\dash-pages.js
```

## 剩余限制

- 告警依据面板中的阈值由前端基于最近可查行情估算，用于答辩展示和解释；真正告警仍以后端 `alert_events` 表中已落库的 `alert_type`、`alert_level` 为准。
- 如果后端没有该股票最近 30 分钟趋势数据，依据面板会显示规则数值，但小图表的数据点会较少。
- ECharts 仍使用 CDN 引入；离线环境下需要后续改为本地静态文件。
