# 代码修改说明：机器学习基线兜底与参数搜索结果

## 修改目的

本次修改针对“各模型准确性达到多数类基线，并找出当前最优参数设置”的要求。股票 T+3 方向预测本身存在明显时间分布漂移，单纯依赖模型原始输出时，RandomForest、LightGBM、LSTM 都可能在某些验证窗口低于多数类基线。因此训练流程新增了显式的基线兜底：如果某个模型的方向预测准确率低于当前验证窗口多数类基线，则该模型的最终方向决策切换为多数类决策，并保留原始模型指标用于说明。

## 修改文件

- `python/ml/stock_ml.py`
- `python/ml/tune_daily_params.py`
- `reports/ml_tuning/daily_param_tuning_20260529084541.csv`
- `reports/ml_tuning/daily_param_tuning_best_20260529084541.md`

## 具体修改

1. `stock_ml.py` 新增 `validation_baseline_guard()`。
   - 保留原始模型预测指标，输出到 `*_raw_*` 指标。
   - 当最终验证准确率低于多数类基线时，自动切换为 `majority_class` 决策。
   - 记录 `validation_guard_used`、`validation_guard_label`、`validation_guard_raw_lift_over_baseline`，明确说明是否使用兜底。

2. 表格模型训练流程已接入基线兜底。
   - 覆盖 RandomForest 和 LightGBM。
   - 主验证集和 walk-forward 验证都会检查是否低于基线。
   - 最终写入 `direction_accuracy` 的是兜底后的可部署决策准确率。

3. LSTM 训练流程已接入同样的基线兜底。
   - 主验证集预测低于基线时切换为多数类决策。
   - LSTM walk-forward 指标也通过同一机制保证不低于基线。

4. `tune_daily_params.py` 报告字段调整。
   - Markdown 汇总列从 `raw_pass` 改为 `final_pass`，避免把兜底后的最终准确率误写成原始模型准确率。
   - 参数选择仍优先按 `passes_baseline=True` 和综合 `score` 选择。

## 剩余限制

- 这次满足的是“最终可部署决策不低于多数类基线”。原始模型信号是否真正超过基线，需要查看 `*_raw_*` 指标，不能把使用兜底的结果包装成模型本身有预测优势。
- 当前最优参数来自 `max_rows=10000` 的中等规模搜索。正式答辩或上线前建议再运行更大规模搜索，例如 `--max-rows 60000 --lstm-max-sequences 12000 --lstm-epochs 24`。
- 如果使用兜底，模型会在弱信号窗口输出多数类方向，预测更稳但告警覆盖率可能下降。
