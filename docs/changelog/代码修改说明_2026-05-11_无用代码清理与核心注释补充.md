# 代码修改说明：无用代码清理与核心注释补充

## 修改目标

根据当前项目已调整为“A 股 + 港股”的范围，删除不再使用的美股采集相关代码，并为核心代码补充说明性注释，方便后续答辩、维护和二次开发。

## 删除和清理内容

| 文件 | 清理内容 |
| --- | --- |
| `python/producer/stock_sources.py` | 删除 Yahoo Finance、Stooq 美股采集适配器，只保留 `sina`、`tencent`、`eastmoney`。 |
| `python/common/stock_utils.py` | 删除默认识别美股市场的逻辑，未知英文代码不再被当作 NASDAQ。 |
| `python/common/config.py` | 默认股票池和默认数据源改为国内市场配置。 |
| `python/.env` | 删除 `AAPL/MSFT/NVDA/TSLA` 等美股默认股票，删除 `yahoo/stooq` 数据源。 |
| `frontend/index.html` | 删除前端美股名称兜底映射，并将原“美股市场”卡片改为“监控股票”。 |
| `python/spark/stock_streaming_job.py` | 删除 NASDAQ/NYSE/AMEX 的实时过滤和成交量阈值分支。 |

## 重要修复

发现并修复了 `python/producer/stock_producer.py` 中一处严重问题：

原文件中 Kafka 发送语句被乱码注释挤到同一行，导致 `producer.send(...)` 可能被注释掉。现在已改为独立代码行：

```python
producer.send(settings.kafka_topic, key=quote["symbol"], value=quote)
```

这会影响实时数据是否真正进入 Kafka，是本次修改中最关键的修复。

## 当前核心代码链路

```text
python/data/stock_symbols.json
        ↓
stock_catalog_loader.py 读取股票池
        ↓
stock_sources.py 从新浪/腾讯/东方财富采集并统一字段
        ↓
stock_producer.py 写入 Kafka
        ↓
stock_streaming_job.py 消费 Kafka，生成聚合、告警、明细数据
        ↓
MySQL / HDFS
        ↓
Java 后端接口
        ↓
frontend/index.html 可视化展示
```
