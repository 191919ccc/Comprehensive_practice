# 机器学习方向判定校准优化说明

## 修改目标

当前页面显示的模型验证指标偏弱，尤其是 `walk_forward_balanced_direction_accuracy` 低于多数类基线。此次优化不继续堆叠复杂模型，而是修正方向预测的最终判定方式：模型输出 UP/DOWN/WATCH 概率后，先在训练期尾部校准最合适的方向阈值，再用于验证、预测和集成，减少简单取最大概率带来的偏置。

## 修改文件

- `python/ml/stock_ml.py`

## 具体修改

1. 增加方向判定校准
   - 新增 `tune_direction_decision_params()`。
   - 使用训练集内部的校准切片搜索 `min_direction_prob` 和 `min_direction_margin`。
   - 优化目标为 `balanced_accuracy + 0.15 * macro_f1`，更关注 UP/DOWN/WATCH 三类是否均衡，而不是只追求原始准确率。

2. 改造方向预测函数
   - `direction_from_probabilities()` 支持 `decision_params`。
   - 当校准结果启用 `direction_gate` 时，只有方向概率足够高且高于 WATCH 概率一定边际，才输出 UP/DOWN，否则输出 WATCH。
   - 如果校准没有带来提升，自动保留原来的 `argmax` 判定方式。

3. 接入普通验证和 walk-forward 验证
   - 表格模型 RF/LGBM 在训练期切出校准集，调好判定阈值后再评估测试集。
   - walk-forward 验证每一折也在训练段内部做校准，再评估未来验证段。
   - 页面优先读取的 `walk_forward_balanced_direction_accuracy` 会在重训后体现校准后的效果。

4. 接入 LSTM
   - LSTM 使用早停验证段生成校准参数。
   - 测试集方向指标和后续预测都会使用该校准参数。

5. 保存和加载校准参数
   - 表格模型新增保存 `*-extra-<version>.joblib`。
   - 预测-only 任务加载模型时会读取校准参数，保证线上预测和训练评估使用一致的方向判定规则。

6. 最佳模型选择更谨慎
   - `direction_quality_score()` 对 walk-forward 低于多数类基线的模型增加扣分。
   - 避免低于基线的模型仅凭局部指标被误判为“最佳”。
