# 代码修改说明：模型排行信号口径统一

## 修改目的

模型页右侧图表已经改为展示 `alert_signal` 的保守信号，但下方“模型优选股票”和“高风险股票”列表仍显示原始 `predicted_signal`，导致页面同时出现“保守信号观望”和“模型信号看多”的冲突。

本次修改统一模型页展示口径：用户看到的主信号一律使用经过置信度过滤后的 `alert_signal`；原始模型方向只作为补充说明。

## 修改文件

- `java-backend/src/main/java/com/recruitment/backend/service/DashboardService.java`
- `frontend/dash-pages.js`

## 具体改动

1. 股票排行接口新增返回 `alert_signal`。
2. 模型优选评分里的模型加分从 `predicted_signal` 改为 `alert_signal`，避免低可信原始看涨信号推高“优选”排名。
3. 后端新增 `raw_signal_distribution`，用于同时返回原始模型方向分布。
4. 模型页图表从单饼图改为“双口径对比”：左侧为最终保守信号，右侧为原始模型方向。
5. 前端排行列表主文案从“模型信号”改为“保守信号”。
6. 当前端发现原始模型信号和保守信号不一致时，补充显示“原始模型：看多/看空”，用于解释模型内部判断和最终展示口径的差异。

## 当前口径

- `predicted_signal`：模型原始方向，用于 drift、回测和模型诊断。
- `alert_signal`：经过置信度和收益幅度过滤后的保守信号，用于前端默认展示和告警参考。

因此，当原始模型给出 UP，但 `alert_signal` 为 WATCH 时，页面最终口径应是“观望”，不是“看多”。
