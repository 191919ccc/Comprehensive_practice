# 代码修改说明：机器学习基线与参数搜索优化

## 修改目的

本次修改围绕两个目标：

1. 统一训练、预测和漂移检测的方向阈值口径，避免模型评估和线上预测使用不同 UP/DOWN/WATCH 标准。
2. 增强调参脚本，让 RandomForest、LightGBM、LSTM 都能按“是否超过多数类基线”筛选最优参数。

## 修改文件

- `python/ml/stock_ml.py`
- `python/ml/train_daily_predict.py`
- `python/ml/predict_daily.py`
- `python/ml/train_predict.py`

## 具体修改

1. `stock_ml.py` 默认方向阈值改为读取 `ML_DIRECTION_THRESHOLD`，不再硬编码 `0.005`。
2. `train_daily_predict.py`、`predict_daily.py`、`train_predict.py` 统一使用 `settings.ml_direction_threshold` 或命令行传入阈值。
3. `model_drift.py` 的真实方向判断阈值同步使用 `ML_DIRECTION_THRESHOLD`，避免漂移检测和训练标签标准不一致。
4. `DashboardService.fetchModelComparison()` 改为读取最新一次训练 `model_version` 的完整指标，避免前端混入不同训练批次的指标。
5. `tune_daily_params.py` 增强为 baseline-gated 搜索：
   - 覆盖 `random_forest`、`lightgbm`、`lstm` 三类模型；
   - 支持 `--direction-thresholds 0.01,0.015,0.02` 多阈值联合搜索；
   - 输出 `passes_baseline`、`direction_lift_over_baseline`、`walk_forward_direction_lift_over_baseline`；
   - 只优先选择主验证和 walk-forward 都超过基线的参数组；
   - 输出 CSV 明细和 Markdown 最优参数摘要。
6. RandomForest 调参脚本使用 `n_jobs=1`，避免 Windows 本地多进程训练触发 `[WinError 5] 拒绝访问`。
7. `tune_direction_decision_params()` 改为基线优先：
   - 先计算校准集多数类基线；
   - 参数候选只有在 accuracy 不低于基线时才优先；
   - 如果模型决策低于基线，则回退到多数类基线决策，并记录 `calibration_lift_over_baseline`。
8. `ensemble_weight_score()` 增加主验证和 walk-forward 基线惩罚：
   - 任一口径明显低于基线时大幅降权；
   - 两个口径都超过基线的模型才正常参与融合。
9. 调参候选组新增：
   - `rf_baseline_guard`：更浅树、更大叶子节点，降低过拟合；
   - `lgbm_baseline_guard`：更小叶子、更强正则，优先稳定超过基线。
10. `daily_param_tuning_best_*.md` 增加 `Selected Parameter Settings`，会直接列出最优参数组的参数。

## 推荐完整参数搜索命令

```powershell
D:\anaconda3\envs\MachineLearn\python.exe -m python.ml.tune_daily_params --models random_forest,lightgbm,lstm --max-rows 60000 --lstm-max-sequences 12000 --lstm-epochs 24 --lstm-patience 6 --direction-thresholds 0.01,0.015,0.02 --out-dir reports\ml_tuning
```

完整搜索完成后，看 `reports/ml_tuning/daily_param_tuning_best_*.md`：

- `passes_baseline=True` 才表示该模型参数在主验证和 walk-forward 两个口径都超过基线；
- 若某个模型仍为 `False`，说明当前数据和特征下该模型还不能稳定超过多数类基线，不应在答辩中声称该模型已优于基线。

## 推荐正式训练命令

当完整搜索确定最优 `direction_threshold` 后，用对应阈值正式训练并写库，例如：

```powershell
D:\anaconda3\envs\MachineLearn\python.exe -m python.ml.train_daily_predict --direction-threshold 0.015 --version-label daily-best
```

如果只想验证指标、不覆盖线上预测：

```powershell
D:\anaconda3\envs\MachineLearn\python.exe -m python.ml.train_daily_predict --direction-threshold 0.015 --version-label daily-best-check --no-write
```

## 剩余限制

- 本轮没有自动跑完整 LSTM 搜索，因为完整搜索耗时较长，应在本机空闲时运行推荐命令。
