# shared-ui 运行时修复与项目复查说明

## 修改目的

继续检查项目时发现 `frontend/shared-ui.js` 的公共工具代码存在运行时风险：部分中文注释和代码在文件中被挤到同一行，`node --check` 只能检查语法，不能发现公共变量或函数被 `//` 注释吞掉的问题。该文件被首页和子页面共同依赖，因此属于前端 P0 风险。

## 修改文件

- `frontend/shared-ui.js`
- `docs/代码修改说明_2026-05-30_shared_ui运行时修复与项目复查.md`

## 具体修改

- 重写 `frontend/shared-ui.js`，保留原有公共能力：
  - `API_BASE`
  - 数值转换和 HTML 转义
  - Markdown 安全渲染
  - API 请求封装
  - 模型指标基线解释
  - 告警触发原因说明
  - 股票历史和导出 URL 构造
- 保证注释和代码分行，避免 `//` 注释吞掉真实代码。

## 复查发现的剩余问题

- 项目实际 Git 根目录是 `E:/`，不是 `E:/作业/ww/Comprehensive_practice`，会导致 `git status` 混入整盘文件。
- `python/.env` 和 `java-backend/src/main/resources/application.yml` 仍包含本机数据库口令或第三方 key。
- `python/models/` 中历史模型文件非常多，总体积约 43GB。
- `frontend/dashboard-home.js` 仍有重复的 `sendAiQuestion()` 定义。
- `frontend/dash-pages.js` 仍保留两段已注释的旧 AI 发送逻辑。
- `python/__pycache__` 缓存目录仍存在。
