# 仪表板可读性与告警交互优化说明

## 修改目标

本次修改针对主仪表板、市场图表中心、模型分析页和风险告警页的展示问题进行优化，重点解决模型指标解释不足、预测方向不直观、图表缺少单位或数值标注、告警分级和忽略反馈不清晰等问题。

## 修改文件

- `frontend/index.html`
- `frontend/dash-pages.js`
- `frontend/dash-pages.css`
- `frontend/alerts.html`

## 具体修改

### 1. 主仪表板

- 在“模型验证状态”卡片中增加进度条、指标名称、多数类基线和高于/低于基线提示。
- 将 AI 预测信号表格改为“标的 / 方向 / 置信度”，方向列使用彩色标签显示 `UP`、`DOWN`、`WATCH`。
- 为实时流量柱状图增加 Y 轴单位“条/分钟”。
- 最新入库行情的涨跌颜色解析增强，支持带 `%`、`+`、`,` 的字符串，避免正涨数据被错误归类。
- 主仪表板告警列表改为前端重新分级，高危只用于“跌幅超过 3% 且伴随成交量异常”的情况；忽略后会半透明显示。

### 2. 市场图表中心

- 市场涨跌堆叠柱状图上涨颜色改为 `#00C896`，下跌颜色改为 `#FF4D4D`。
- 图例增加上涨/下跌数量和百分比。
- 行业异常柱状图增加柱顶数值标签。
- 最新入库行情使用增强后的涨跌解析逻辑，正涨显示绿色，负涨显示红色。

### 3. 模型分析页

- 综合信号分布图改为甜甜圈图，并在中心直接显示“观望 / 看多 / 看空”数量。
- 模型优选股票中的 `NONE` 改为“暂无预测”，并加 tooltip 说明“当前模型未对该股票生成方向信号”。
- 模型对比结果增加说明文字，明确当前模型指标是否低于多数类基线。

### 4. 风险告警页

- 增加快速筛选 tab：全部、价格异动、成交量、模型告警。
- 告警卡片按重新计算后的高危/中危/低危显示不同背景和标签。
- 点击“忽略”后，告警卡片立即半透明化，刷新后仍根据 `IGNORED` 状态保持弱化显示。

## 验证命令与结果

已执行：

```powershell
node --check frontend\dash-pages.js
```

结果：通过，无 JavaScript 语法错误。

已执行主仪表板内联脚本抽取后的语法检查：

```powershell
$html = Get-Content -Raw -Encoding UTF8 frontend\index.html
$m = [regex]::Match($html, '<script>([\s\S]*)</script>')
$tmp = Join-Path $env:TEMP ('index-inline-' + [guid]::NewGuid().ToString() + '.js')
Set-Content -LiteralPath $tmp -Encoding UTF8 -Value $m.Groups[1].Value
node --check $tmp
Remove-Item -LiteralPath $tmp
```

结果：通过，无 JavaScript 语法错误。

## 后续注意

- 本次只修改前端展示与交互逻辑，没有重新训练模型，也没有修改模型训练代码。
- 告警分级目前在前端做展示层重算，如果后续希望数据库中的 `alert_level` 也一致，需要再同步修改后端告警生成逻辑。
- 图表效果需要在后端服务运行时打开页面做一次浏览器视觉确认。
