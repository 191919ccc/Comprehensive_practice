# 代码修改说明：WATCH 边界调整为 1.5%

## 1. 修改目标

本次修改的目标是把三分类方向标签的 `WATCH` 边界固定为 `1.5%`，让模型训练时按更明确的涨跌幅定义标签：

```text
未来 3 日收益率 > +1.5%  -> UP
未来 3 日收益率 < -1.5%  -> DOWN
其他                     -> WATCH
```

这样可以减少小幅波动样本对 UP / DOWN 的干扰，更符合“股票价格预测与告警”项目定位：只有未来波动达到一定幅度时才触发方向告警。

## 2. 修改文件

修改文件：

- `python/.env`
- `python/common/config.py`
- `python/ml/train_daily_predict.py`

新增文档：

- `docs/代码修改说明_2026-05-09_WATCH边界调整为1.5pct.md`

## 3. 具体代码修改

### 3.1 新增环境配置

在 `python/.env` 中新增：

```text
ML_DIRECTION_THRESHOLD=0.015
```

这里的 `0.015` 就是 `1.5%`。

### 3.2 配置类读取阈值

在 `python/common/config.py` 中新增：

```python
ml_direction_threshold = float(os.getenv("ML_DIRECTION_THRESHOLD", "0.015"))
```

这样训练脚本可以从统一配置读取方向边界。

### 3.3 训练入口默认使用固定阈值

在 `python/ml/train_daily_predict.py` 中，将 `--direction-threshold` 的默认值改为：

```python
default=stock_ml.settings.ml_direction_threshold
```

因此以后直接执行：

```powershell
D:\anaconda3\envs\MachineLearn\python.exe -m python.ml.train_daily_predict
```

就会默认使用 `1.5%` 固定边界，不需要每次手动追加 `--direction-threshold 0.015`。

## 5. 预期影响

调整后，`WATCH` 类样本会增加，UP / DOWN 会变成更强波动样本。预期效果是：

1. UP / DOWN 标签更明确；
2. 方向告警更保守；
3. 普通 accuracy 可能提升；
4. balanced accuracy 不一定立刻提升，需要看三类 recall 是否更均衡；
5. 前端预测信号会更偏向 WATCH，只有更明显的涨跌才会输出 UP 或 DOWN。

## 6. 注意事项

如果训练后模型几乎全预测 WATCH，说明 `1.5%` 对当前 3 日 horizon 可能偏高，可以回调到：

```text
0.010
```

也就是 `1.0%`。如果 UP / DOWN 仍然太多、告警太频繁，再继续提高阈值。
