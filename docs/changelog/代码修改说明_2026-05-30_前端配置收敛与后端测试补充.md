# 前端配置收敛与后端测试补充说明

## 修改目的

本次修改解决几个后续维护风险：
1. 前端 API 地址分散在多个脚本中，部署或换端口时容易漏改。
2. 首页保留了一套 AI Markdown 清洗兜底逻辑，和 `shared-ui.js` 存在重复维护风险。
3. `vendor-loader.js` 使用 `document.write`，脚本加载方式不够稳。
4. 子页面 CSS 依赖 Google Fonts，断网答辩时字体请求会失败。
5. Java 后端缺少业务单元测试，CI 只能证明能编译，不能证明核心方法行为稳定。

## 具体修改

1. `shared-ui.js` 统一负责 API 地址兜底，支持通过 `window.STOCK_API_BASE` 或 `window.API_BASE` 覆盖。
2. `dashboard-home.js` 和 `dash-pages.js` 不再各自硬编码后端地址，只读取 `StockShared.API_BASE`。
3. 首页 AI Markdown 输出格式不变，仍由 `StockShared.renderMarkdown()` 执行 `marked.parse()` 和 HTML 清洗；删除首页重复的清洗实现。
4. `vendor-loader.js` 改为动态创建 `<script>`，本地 vendor 失败时再回退 CDN，并让页面业务脚本等 vendor 加载完成后再执行。
5. 删除 `dash-pages.css` 的 Google Fonts 远程导入，改用系统字体栈。
6. 新增 `DashboardServiceTest`，覆盖告警状态更新、历史 CSV 导出转义、告警统计结构。
7. GitHub Actions 后端步骤由只编译改为执行 `mvn -q test`。

## 验证命令

```powershell
node --check frontend\vendor-loader.js frontend\shared-ui.js frontend\dashboard-home.js frontend\dash-pages.js
D:\anaconda3\envs\MachineLearn\python.exe -m unittest discover -s python\tests -p "test_*.py"
cd java-backend
mvn -q test
```

## 剩余限制

- AI Markdown 的展示格式按要求未改；后续如果要改渲染样式，应只改 `shared-ui.js` 中的统一渲染逻辑。
- 前端页面仍保留 CDN 兜底，只是不再依赖公网字体；如果答辩环境必须完全离线，可以进一步移除 CDN fallback。
