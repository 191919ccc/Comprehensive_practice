# 代码修改说明：实时股票池与名称显示修正

## 修改目标

解决前端下拉框中部分 A 股只显示 `000001 · 000001`、`000100 · 000100`，以及用户误以为这些股票没有实时爬取数据的问题。

## 问题原因

1. `python/data/stock_symbols.json` 文件内容损坏，中文字段出现乱码并导致 JSON 解析失败，实时采集程序可能回退到 `.env` 中较小的股票池。
2. 前端 `collectStocks()` 优先使用 `ml_predictions`，而历史日线预测表中部分股票的 `company_name` 等于股票代码，导致后续实时行情中的真实名称没有机会覆盖旧名称。
3. 美股实时数据量少不是同一个问题。美股当前主要依赖 Yahoo Finance 的 1 分钟日内数据和 Eastmoney/Tencent 兜底快照，在非美股交易时段不会持续产生大量新分钟行情。

## 修改文件

| 文件 | 修改内容 |
| --- | --- |
| `python/data/stock_symbols.json` | 重建为合法 JSON，扩充到 119 只股票，补齐 A 股、港股和美股股票的名称、市场、行业和分类。 |
| `frontend/index.html` | 调整股票下拉框合并顺序，优先显示实时行情、重点榜和风险榜中的股票名称；增加常用股票名称映射，避免旧预测数据只显示代码。 |

## 具体修改

1. 重建实时采集股票池配置，补入 `000001`、`000002`、`000063`、`000100`、`000166`、`000338` 等此前只出现在日线预测中的 A 股代码。
2. 前端股票选择框改为按数据可靠性合并：
   - `latest_ticks`
   - `focus_stocks`
   - `risk_stocks`
   - `optimal_stocks`
   - `ml_predictions`
3. 当前端遇到 `company_name` 等于股票代码的旧数据时，使用本地 `KNOWN_SYMBOL_NAMES` 映射补充显示名称。
4. 对仅来自机器学习预测的数据，在下拉框追加“日线预测”提示，避免与实时分钟行情混淆。


结果显示股票池可正常加载，共 119 只股票，`000100` 可识别为 `TCL Technology`。


结果为 `inline script ok`。

数据库检查显示，`000001`、`000002`、`000063`、`000100`、`000166` 已经存在实时行情记录，例如 `000100` 来源为 `sina`，名称为 `TCL科技`。

## 运行说明

修改后需要重启真实采集和后端服务，让新的股票池配置生效：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\stop_demo.ps1
powershell -ExecutionPolicy Bypass -File .\tools\start_demo.ps1 -UseRealCrawler
```

如果浏览器仍显示旧下拉框内容，刷新页面或清理浏览器缓存后再访问：

```text
http://127.0.0.1:5500/index.html
```

## 剩余说明

美股数据少主要受交易时段和数据源限制影响。当前项目的美股分钟线主源是 Yahoo Finance 的 `1m / 1d` 接口，非交易时段通常只返回最近交易日的分钟数据，不会像 A 股在交易时段一样持续新增。若要增强美股实时展示，需要在美股交易时段运行采集程序，或接入更稳定的付费实时行情源。
