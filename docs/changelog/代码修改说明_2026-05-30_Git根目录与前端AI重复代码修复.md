# Git 根目录与前端 AI 重复代码修复说明

## 修改目的

本次修改解决两个维护风险：

1. 项目没有独立 Git 仓库，`git status` 会从 `E:/` 根目录扫描整盘文件，导致版本管理不可用。
2. 前端 AI 发送逻辑存在重复实现，首页有两个同名 `sendAiQuestion()`，子页面保留两段废弃注释代码，后续维护容易改错位置。

## 修改文件

- `.gitignore`
- `frontend/dashboard-home.js`
- `frontend/dash-pages.js`
- `docs/代码修改说明_2026-05-30_Git根目录与前端AI重复代码修复.md`

## 具体修改

1. 初始化项目独立 Git 仓库
   - 在 `E:/作业/ww/Comprehensive_practice` 执行 `git init`。
   - 当前 `git rev-parse --show-toplevel` 已指向项目目录，不再指向 `E:/`。

2. 新增项目级 `.gitignore`
   - 忽略 Python 缓存、虚拟环境、本机 `.env`。
   - 忽略 `python/models/`、`*.joblib`、`*.pt`、`*.pth` 等训练产物。
   - 忽略 Java `target/`、日志、`.runtime/`、IDE 配置和系统元数据。

3. 删除前端重复 AI 代码
   - `frontend/dashboard-home.js` 删除旧版 `sendAiQuestion()`，保留带 `aiChatHistory` 的新版上下文对话实现。
   - `frontend/dash-pages.js` 删除两段已注释的旧 `sendAiQuestion()` 实现，只保留实际运行函数。
