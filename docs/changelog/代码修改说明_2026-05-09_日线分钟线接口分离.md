# 日线与分钟线接口分离修改说明

## 修改目标

解决主仪表板中“日线+预测”看起来没有真实日线数据、只能显示预测兜底图的问题，并明确区分：

- 实时按钮：使用分钟线接口。
- 日线+预测按钮：使用日线接口。

## 问题原因

项目已经拆分了日线表：

- `daily_stock_bars`：股票日线数据
- `daily_index_bars`：指数日线数据
- `price_ticks`：实时或分钟级行情数据

但是后端 `/api/stocks/{symbol}/daily` 接口仍然在旧表 `price_ticks` 中查询：

```sql
WHERE source = 'akshare_cold_start'
```

你之前已经把 `akshare_cold_start` 从 `price_ticks` 中删除，日线数据迁移到了 `daily_stock_bars`。所以日线接口查不到数据，前端只能进入兜底展示。

## 修改文件

- `java-backend/src/main/java/com/recruitment/backend/service/DashboardService.java`
- `frontend/index.html`

## 具体修改

### 1. 后端日线接口改查新日线表

`getStockDailyData` 现在优先查询：

```sql
daily_stock_bars
```

返回字段包括：

- `trade_date`
- `open`
- `high`
- `low`
- `close`
- `change_pct`
- `volume`
- `turnover`
- `source`

### 2. 日线表无数据时做降级

如果 `daily_stock_bars` 中没有该股票，例如部分港股或美股没有冷启动日线，则后端再从 `price_ticks` 按日期聚合，生成日级数据。

此时返回：

```json
"daily_source": "price_ticks_aggregated"
```

如果来自真正日线表，则返回：

```json
"daily_source": "daily_stock_bars"
```

### 3. 前端显示数据来源

主图标题和状态标签现在会区分：

- `日线历史与AI预测`
- `分钟线聚合日线与AI预测`

这样用户能看出当前图表用的是正式日线，还是实时分钟线聚合出来的日线。

## 当前接口分工

```text
实时按钮
GET /api/stocks/{symbol}/trend?minutes=1440
数据源：price_ticks

日线+预测按钮
GET /api/stocks/{symbol}/daily?days=120
优先数据源：daily_stock_bars
兜底数据源：price_ticks 按日期聚合
```

## 后续注意

- 如果想让港股、美股也有真正历史日线，需要把对应市场的日线数据也写入 `daily_stock_bars` 或新建跨市场日线表。
- 本次不需要重新训练模型。
- 修改后需要重启 Java 后端，前端刷新后接口才会生效。
