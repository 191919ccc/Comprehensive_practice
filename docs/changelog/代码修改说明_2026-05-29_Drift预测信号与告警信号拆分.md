# Drift 预测信号与告警信号修复说明

## 修改目的

本次目标是解决 `drift_check=True` 的真实失配问题。上一步已经确认漂移检测不再被告警过滤污染，但最近 50 条里实际方向主要是 DOWN，而模型最终 `predicted_signal` 偏 WATCH/UP。

进一步核查后发现：`predicted_next_price` 反推出来的 `predicted_return` 多数已经是负收益，但最终方向信号只按分类概率输出，导致“价格预测向下，方向信号却不是 DOWN”的不一致。

## 修改文件

- `python/common/config.py`
- `python/ml/stock_ml.py`

## 具体修改

1. 新增配置 `ML_PREDICTION_RETURN_SIGNAL_THRESHOLD`，默认值为 `0.003`。
2. 在 `stock_ml.py` 中新增 `apply_return_signal_override()`。
3. 当 `predicted_return >= 0.003` 时，最终 `predicted_signal` 校正为 `UP`。
4. 当 `predicted_return <= -0.003` 时，最终 `predicted_signal` 校正为 `DOWN`。
5. 保留 `alert_signal` 作为告警过滤后的保守信号，不再污染 drift 评估用的 `predicted_signal`。
6. 在模型指标中写入 `prediction.return_signal_override_threshold=0.003`，方便后续说明和追踪。

## 剩余限制

这次修复解决的是最终信号和价格回归预测不一致的问题，并让 drift 检测恢复通过。它不是证明模型已经很强。当前 LSTM 的 walk-forward 仍然不稳定，后续如果继续优化模型质量，重点应放在：

- 下跌样本识别稳定性；
- walk-forward 场景下的 LSTM 泛化；
- 保存更完整的概率和收益解释字段，便于前端展示预测依据。
