# 代码修改说明：T+5 目标与模型性能优化

## 修改目的

本次修改用于验证并落地“预测未来 5 个交易日风险方向”的训练方案。之前 T+3 的 LightGBM walk-forward lift 约为 `+0.0041`，提升较小；T+5 标签分布更均衡，因此先按 T+5 正式训练并与 T+3 基线比较。

## 修改文件

- `python/common/config.py`
- `python/ml/stock_ml.py`
- `python/ml/predict_daily.py`
- `python/.env.example`
- `frontend/index.html`
- `frontend/dashboard-home.js`

## 主要改动

1. 默认训练目标改为 T+5：
   - `ML_PREDICTION_HORIZON=5`
   - `ML_DIRECTION_THRESHOLD=0.020`
   - `predict_daily.py` 不再硬编码 T+3，而是读取配置里的 `ml_prediction_horizon`。

2. LightGBM 参数调整为更稳的 T+5 配置：
   - `n_estimators=500`
   - `learning_rate=0.02`
   - `num_leaves=31`
   - `min_child_samples=80`
   - `reg_lambda=5.0`
   - 保留 `class_weight="balanced"`。

3. LSTM 降低复杂度，避免 walk-forward 退化：
   - `window=30`
   - `hidden=48`
   - `layers=1`
   - `dropout=0.40`
   - `lr=0.0003`

4. 新增金融特征：
   - 近 5/10/20 日最大回撤。
   - 行业内 5/10/20 日收益 percentile 排名。
   - 行业 3/10/20 日动量与相对收益。
   - 上证、创业板、沪深 300 的 5/10 日波动率。
   - 市场指数 5/10 日波动率。
   - 连续放量特征。
   - `return_3 * volume_ratio_20_raw` 等价量交互项。

5. 告警置信度阈值调整：
   - `ML_ALERT_UP_CONFIDENCE_THRESHOLD=0.64`
   - `ML_ALERT_DOWN_CONFIDENCE_THRESHOLD=0.75`
   - 目标是让 LightGBM 告警回测覆盖率落在 `2%~5%`，避免 `0.55/0.70` 带来的过高覆盖率。

6. 前端口径同步：
   - 首页主图文案从 `T+3` 改为 `T+5`。
   - 兜底预测图标签从 `T+3预测` 改为 `T+5预测`。

## 验证结果

最新训练版本：

```text
daily-h5-bestparams-h5-20260601111012
```

LightGBM 核心指标：

| 指标 | T+3 基线 | 本次 T+5 |
| --- | ---: | ---: |
| walk_forward_direction_lift_over_baseline | 0.0041 | 0.0063 |
| walk_forward_balanced_direction_accuracy | 0.4039 | 0.4125 |
| walk_forward_direction_macro_f1 | 0.4030 | 0.4105 |
| alert_backtest_coverage | 0.0042 | 0.0306 |
| alert_backtest_hit_rate | 0.6200 | 0.4305 |
| alert_backtest_up_avg_return | 0.0304 | 0.0186 |
| alert_backtest_down_avg_inverse_return | 0.0000 | 0.0308 |

结论：T+5 的 walk-forward lift、balanced accuracy、macro F1 均高于原 T+3 基线，且 LightGBM 告警覆盖率提升到 `3.06%`，位于目标 `2%~5%` 区间，因此保留本次修改，不回退。

## 剩余限制

- LSTM 的普通验证集 lift 为正，但 walk-forward 仍被基线保护回 WATCH，不能作为主力模型宣传；前端和答辩口径仍应以 LightGBM 为主。
- 最新 92 条日线预测因为置信度过滤较保守，`alert_signal` 仍可能全部为 WATCH；这属于告警侧保守策略，不影响 `predicted_signal` 的模型评估口径。
- T+5 drift 需要等待未来 5 个交易日真实日线数据，刚训练完成时会显示 `not_enough_evaluable_predictions`，这是正常现象。
- 特征构造阶段出现 `DataFrame is highly fragmented` 性能警告，当前不影响训练结果，后续可以通过批量 `concat` 优化特征拼接性能。
