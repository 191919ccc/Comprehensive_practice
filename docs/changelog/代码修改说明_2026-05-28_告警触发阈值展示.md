# 代码修改说明：告警触发阈值展示

## 修改目的

本次修改是为了解决风险告警卡片只显示告警结果、没有解释触发依据的问题。现在用户看到一条告警时，可以直接看到触发规则、涨跌幅是否超过阈值、成交量是否超过阈值，不需要再展开额外图表判断原因。

## 修改文件

- `python/spark/stock_streaming_job.py`
- `python/sql/init.sql`
- `java-backend/src/main/java/com/recruitment/backend/service/DashboardService.java`
- `frontend/dash-pages.js`
- `frontend/dash-pages.css`
- `docs/代码修改说明_2026-05-28_告警触发阈值展示.md`

## 具体修改

1. `build_alerts()` 写入 `alert_events` 时新增：
   - `price_threshold`：价格波动告警阈值；
   - `volume_threshold`：成交量告警阈值。
2. `init.sql` 的 `alert_events` 表新增同名字段，保证新环境初始化时表结构完整。
3. `DashboardService.ensureExtendedTables()` 启动时自动尝试给旧表补充 `price_threshold` 和 `volume_threshold` 字段。
4. `/api/dashboard` 的 `latest_alerts` 查询和 `/api/alerts` 的筛选查询都返回这两个阈值字段。
5. `dash-pages.js` 新增 `buildAlertReason()`，在每条告警卡片中直接显示：
   - 触发规则；
   - 当前涨跌幅；
   - 价格阈值；
   - 当前成交量；
   - 成交量阈值。
6. 移除告警卡片中的“查看依据”按钮入口，避免用户还需要额外展开图表才能理解原因。

## 解决的问题

- 用户可以一眼看到“为什么告警”。
- 告警说明来自后端落库阈值，而不是前端临时估算。
- 价格波动、成交量异动、价格+成交量双异动三类规则都有清晰文案。

## 数据库注意事项

后端启动时会自动执行字段补齐。如果 Spark 流任务在后端重启前已经开始写入新字段，需要先手动执行：

```sql
ALTER TABLE alert_events
ADD COLUMN price_threshold DECIMAL(8,2) DEFAULT 0,
ADD COLUMN volume_threshold BIGINT DEFAULT 0;
```

## 验证命令

```powershell
python -m py_compile python\spark\stock_streaming_job.py
node --check frontend\dash-pages.js
cd java-backend
mvn -q -DskipTests compile
```

## 剩余限制

- 历史旧告警的阈值字段默认是 0，只有新产生的告警会写入真实阈值。
- 模型漂移类告警没有价格/成交量阈值，前端会显示为模型风险信号触发。
