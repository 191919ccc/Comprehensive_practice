# 代码修改说明：AKShare 日线数据拆表

## 1. 修改目标

原来 `price_ticks` 同时保存实时行情、AKShare 股票日线、AKShare 指数日线和其他来源数据，长期会导致训练数据、展示数据、实时流数据混在一起。

本次修改将 AKShare 历史日线拆成独立表：

- `daily_stock_bars`：股票日线历史数据，用于机器学习训练。
- `daily_index_bars`：指数日线历史数据，用于指数特征和市场特征。
- `price_ticks`：继续保留为实时行情明细表。

## 2. 修改文件

- `python/ml/daily_bar_store.py`
- `python/ml/cold_start.py`
- `python/ml/stock_ml.py`
- `python/ml/train_daily_predict.py`
- `python/ml/predict_daily.py`
- `python/ml/model_drift.py`
- `python/sql/init.sql`
- `python/sql/split_daily_bars.sql`

## 3. 具体修改

### 3.1 新增日线存储模块

新增 `python/ml/daily_bar_store.py`，集中管理：

- 日线股票表名：`daily_stock_bars`
- 日线指数表名：`daily_index_bars`
- AKShare 股票来源：`akshare_cold_start`
- AKShare 指数来源：`akshare_index`
- 建表 SQL
- 日线字段列表
- `source -> table` 映射

这样 `cold_start.py` 和 `stock_ml.py` 不再各自硬编码日线表结构。

### 3.2 cold_start 写入新表

`python/ml/cold_start.py` 原来把 AKShare 数据写入 `price_ticks`。

现在改为：

- `source = akshare_cold_start` 写入 `daily_stock_bars`
- `source = akshare_index` 写入 `daily_index_bars`

同时新增 `trade_date` 字段，用于 `(symbol, trade_date, source)` 去重。

冷启动命令不变，例如：

```powershell
python -m python.ml.cold_start --days 2500 --sources tencent --sleep 0
```

指数补齐命令也不变：

```powershell
python -m python.ml.cold_start --include-index --index-only --days 2500 --sources tencent --sleep 0
```

区别是以后数据会进新表，而不是继续混进 `price_ticks`。

### 3.3 训练读取新表

`python/ml/stock_ml.py` 中的 `load_price_ticks(source=...)` 保持函数名不变，但内部逻辑调整：

- `source="akshare_cold_start"` 时优先读 `daily_stock_bars`
- `source="akshare_index"` 时优先读 `daily_index_bars`
- 如果新表为空，会回退读取旧 `price_ticks` 中对应 `source` 的数据

这样做是为了兼容你现在数据库里已经存在的旧数据，不会因为新表刚创建为空就导致训练不能跑。

### 3.4 初始化 SQL 增加新表

`python/sql/init.sql` 已新增：

- `daily_stock_bars`
- `daily_index_bars`

以后新建数据库时会自动包含拆分后的表结构。

### 3.5 新增旧数据迁移 SQL

新增 `python/sql/split_daily_bars.sql`，用于把旧 `price_ticks` 里的 AKShare 日线搬到新表：

- `price_ticks.source = 'akshare_cold_start'` -> `daily_stock_bars`
- `price_ticks.source = 'akshare_index'` -> `daily_index_bars`

该脚本默认不删除旧数据，只做创建和复制，避免误删。

## 5. 验证 SQL

迁移后检查新表数据：

```sql
USE stock_stream;

SELECT COUNT(*) AS stock_rows FROM daily_stock_bars;
SELECT COUNT(*) AS index_rows FROM daily_index_bars;

SELECT source, COUNT(*) AS rows_count
FROM price_ticks
GROUP BY source
ORDER BY rows_count DESC;
```

检查每只股票的日线范围：

```sql
SELECT symbol, COUNT(*) AS rows_count, MIN(event_time), MAX(event_time)
FROM daily_stock_bars
GROUP BY symbol
ORDER BY symbol;
```

检查指数范围：

```sql
SELECT symbol, COUNT(*) AS rows_count, MIN(event_time), MAX(event_time)
FROM daily_index_bars
GROUP BY symbol
ORDER BY symbol;
```

## 6. 可选清理旧表

确认 `daily_stock_bars` 和 `daily_index_bars` 数据没问题后，才可以考虑清理 `price_ticks` 中旧的 AKShare 日线：

```sql
DELETE FROM price_ticks
WHERE source IN ('akshare_cold_start', 'akshare_index');
```

这个操作是删除旧表里的重复日线数据，不影响新表。建议先备份或确认新表行数正确后再执行。

## 9. 剩余注意事项

- `price_ticks` 仍然保留给实时行情和 Spark 流处理使用。
- `daily_stock_bars` 和 `daily_index_bars` 是机器学习历史日线专用表。
- 旧 `price_ticks` 中的 AKShare 数据不会自动删除，需要你确认迁移成功后手动清理。
