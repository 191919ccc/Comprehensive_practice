# 下跌风险目标收紧修改说明

## 修改目的

上一版 `downside_risk` 已经让模型开始识别 `DOWN / WATCH`，但轻量验证显示风险标签偏松：

- LSTM `walk_forward_predicted_down_ratio` 达到约 72.7%，看空覆盖过高；
- LightGBM 的 DOWN 告警命中率偏低，且部分 DOWN 告警后的平均收益为正；
- 风险目标需要更强调“实质性下跌风险”，而不是宽泛的短期波动。

本次在保留风险识别方向的前提下收紧标签和告警过滤。

## 修改文件

- `python/common/config.py`
- `python/.env.example`
- `python/.env`（本地忽略文件）
- `python/ml/stock_ml.py`

## 关键改动

1. 新增 `ML_RISK_DOWNSIDE_THRESHOLD`
   - 默认值从隐含 2% 风险改为显式 `0.030`；
   - 风险标签至少要求未来 N 日出现 3% 级别回撤。

2. 提高高波动风险阈值
   - `ML_RISK_VOLATILITY_THRESHOLD` 从 `0.040` 提高到 `0.060`；
   - 高波动条件下还要求出现至少 `风险阈值 * 0.7` 的回撤。

3. 新增 `ML_RISK_ALERT_MIN_RETURN`
   - 默认 `0.010`；
   - DOWN 告警必须满足预测收益至少低于 `-1%`，避免小幅负收益也放行告警。

4. 压制过度看空模型的集成权重
   - 当 `predicted_down_ratio` 超过 42%、50%、60% 时逐级降低该模型权重；
   - 目标是避免 LSTM 这类高召回模型把集成结果推成“满屏看空”。

5. 增加指标记录
   - `risk_downside_threshold`
   - `risk_volatility_threshold`
   - `risk_alert_min_return`

## 判断标准

比上一版更理想的结果应该是：

- `predicted_down_ratio` 明显低于 70%，最好在 25% 到 50%；
- `down_recall` 不要掉到 30% 以下；
- `alert_backtest_down_hit_rate` 高于上一版；
- `alert_backtest_down_avg_inverse_return` 为正，且幅度不能接近 0；
- 集成权重不再过度偏向高覆盖的 LSTM。

## 回退方式

如果这版过严导致 DOWN 识别不足，可以临时放宽：

```bat
set ML_RISK_DOWNSIDE_THRESHOLD=0.020
set ML_RISK_VOLATILITY_THRESHOLD=0.040
set ML_RISK_ALERT_MIN_RETURN=0.000
```

如果要撤销本次代码提交：

```bat
git revert <本次提交ID>
git push origin main
```

## 剩余限制

- 本次只调整标签和过滤口径，最终是否更好必须看重新训练结果。
- 如果正式训练后 DOWN 覆盖仍过高，下一步应把 LSTM 的风险模式决策阈值单独提高，而不是继续全局收紧标签。
