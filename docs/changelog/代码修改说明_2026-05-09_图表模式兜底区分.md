# 图表模式兜底区分修改说明

## 修改目标

解决主仪表板中“实时”和“日线+预测”两个按钮切换后图表和时间轴看起来一样的问题。

## 问题原因

原逻辑中，`日线+预测` 模式会请求：

```text
/api/stocks/{symbol}/daily?days=120
```

如果该股票没有日线历史数据，前端会静默回退到分钟线接口：

```text
/api/stocks/{symbol}/trend?minutes=1440
```

所以用户看到的仍然是分钟时间轴，例如 `05-09 21:41`、`05-09 21:47`，导致“实时”和“日线+预测”两个模式显示一样。

另外，实时模式在某只股票没有分钟线时，也可能使用 dashboard 全局趋势数据兜底，容易造成当前股票和图表不对应。

## 修改文件

- `frontend/index.html`

## 具体修改

### 1. 新增预测兜底图

新增 `renderPredictionOnlyChart(data, main, reason)`：

- 当没有日线历史时，不再回退到分钟线。
- 显示“当前 / T+3预测”两点图。
- 图表标题显示原因，例如“暂无日线历史，显示AI预测”。
- 同步更新右上角状态标签。

### 2. 区分实时与日线模式

`renderSelectedChart` 新逻辑：

- `日线+预测` 模式：
  - 有日线历史：显示日线收盘价 + AI 预测。
  - 没有日线历史：显示“当前 / T+3预测”兜底图。
  - 不再静默回退到分钟线。
- `实时` 模式：
  - 有分钟线：显示分钟趋势。
  - 没有分钟线：显示“当前 / T+3预测”兜底图。

### 3. 避免全局趋势误用

`renderPriceChart` 不再在指定股票没有分钟线时自动使用 dashboard 全局趋势，避免不同股票显示同一条分钟线。

## 验证方式

```powershell
$html = Get-Content -Raw -Encoding UTF8 frontend\index.html
$m = [regex]::Match($html, '<script>([\s\S]*)</script>')
$tmp = Join-Path $env:TEMP ('index-inline-' + [guid]::NewGuid().ToString() + '.js')
Set-Content -LiteralPath $tmp -Encoding UTF8 -Value $m.Groups[1].Value
node --check $tmp
Remove-Item -LiteralPath $tmp
```

## 后续注意

- 如果要让美股也显示真正日线，需要后端或数据入库流程补齐美股日线表。
- 本次只修改前端图表模式选择逻辑，不需要重新训练模型。
