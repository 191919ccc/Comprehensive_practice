# 数据清洗到模型训练重构说明

## 修改目的

正式训练后模型出现了两个明显问题：原始模型大量输出 DOWN，但告警过滤又全部压成 WATCH；同时训练数据质量没有形成统一门禁，OHLC 异常、缺失元数据和异常涨跌幅可能影响波动率、量价和行业相对强弱特征。本次先重构数据清洗到训练决策这条链路，让后续训练结果更稳定。

## 修改文件

- `python/common/stock_utils.py`
- `python/producer/stock_sources.py`
- `python/ml/cold_start.py`
- `python/spark/stock_streaming_job.py`
- `python/ml/stock_ml.py`
- `python/ml/train_daily_predict.py`

## 核心改动

1. 新增统一清洗规则：
   - `market_change_limit()`：按市场设置数据质量涨跌幅上限。
   - `is_valid_ohlc()`：检查 `open/high/low/close/previous_close/volume`。
   - `normalize_quote_text()`：补齐 `company_name/category/sector/market`，避免分类特征空值漂移。

2. 日线冷启动入库前加强校验：
   - A 股日线继续使用 `is_valid_tick()`。
   - 指数日线也改为统一合法性校验，不再只看 `last_price > 0`。
   - 写入前会过滤 OHLC 不一致、成交量异常、涨跌幅异常的行。

3. Spark 实时流入库前加强过滤：
   - 增加 `event_time`、`symbol`、OHLC、`previous_close` 校验。
   - `company_name/category/sector` 空值会被补齐。
   - 港股异常涨跌幅上限从 50% 收紧到 30%。
   - 增加美股/纳斯达克/纽交所兜底上限 25%。

4. ML 训练前增加强清洗：
   - 在特征工程前调用 `clean_price_frame()`。
   - 过滤异常时间、空代码、非正价格、非正成交量、OHLC 逻辑错误、异常涨跌幅。
   - 按 `symbol + event_time + source` 去重。
   - 训练日志会输出清洗前后行数和剔除原因。

5. 训练指标记录数据质量：
   - 新增 `data_quality_metrics()`。
   - 将输入行数、输出行数、剔除行数、OHLC 异常、成交量异常、涨跌幅异常、重复行等写入 `ml_model_metrics`。

6. 风险模型决策更稳：
   - `tune_direction_decision_params()` 从偏召回改为更重视下跌精度。
   - DOWN 覆盖率按真实下跌比例设置上限，避免完整训练后大面积预测 DOWN。
   - 搜索更高的最小方向概率和方向边际，减少过度看空。
   - 高风险评分且证据足够时，允许保留 DOWN，不再被 `predicted_return > -1%` 一刀切压成 WATCH。
7. 训练准备链路拆分为“特征工程”和“监督标签构造”：
   - 新增 `prepare_feature_frame()`，清洗、选源、去重和技术特征只做一次。
   - 新增 `build_supervised_dataset()`，不同预测周期和阈值只重新构造标签。
   - `prepare_model_datasets()` 和 `horizon_experiment_metrics()` 复用同一份特征表，避免重复跑完整特征工程。
   - 标签列批量拼接，消除 `DataFrame is highly fragmented` 警告。
8. 新增训练质量门禁摘要：
   - 训练结束后打印 `quality_gate=PASS/FAIL`。
   - 关键指标包括 `lightgbm_walk_forward_lift`、`lightgbm_walk_forward_down_precision`、`lightgbm_walk_forward_predicted_down_ratio`、`ensemble_lightgbm_weight`、`training_data_drop_ratio`。
   - 同一组门禁指标会写入 `ml_model_metrics` 的 `quality_gate` 分组，方便前端或答辩时直接证明训练是否达标。
   - 风险目标模式下，评估入口会把 UP 预测统一归并为 WATCH，避免“下跌风险识别”任务继续混入看多类别。
9. 新增正式训练后只读验证脚本：
   - `python/ml/verify_training_quality.py`
   - 从 MySQL 读取最新 `ml_model_metrics` 和 `ml_predictions`。
   - 校验 `quality_gate.passed`、LightGBM walk-forward lift、DOWN precision、DOWN 覆盖率、ensemble 权重和训练数据丢弃率。
   - 验证失败时返回非 0 退出码，避免旧模型或坏模型被误认为已经达标。
10. 正式训练写库前增加硬门禁：
   - `train_daily_predict.py` 在非 `--no-write` 模式下默认要求 `quality_gate.passed = 1.0`。
   - 如果质量门禁失败，会在保存模型和写 MySQL 之前退出，避免坏模型覆盖当前库。
   - 只有调试时显式加 `--allow-quality-gate-fail` 才允许失败模型继续保存/写库。
11. 更新 README 训练入口：
   - README 中旧的 `daily-best` 训练命令已替换为 `clean-risk` 风险识别训练命令。
   - README 新增正式训练后的 `verify_training_quality --version-prefix clean-risk` 验证步骤。
   - README 明确说明当前模型输出应按“风险/观望”理解，不再按传统看多/看空三分类解释。
12. 新增一键正式训练与验证脚本：
   - `tools/train_clean_risk.ps1`
   - 默认先跑轻量 `--no-write` 检查，再跑正式训练，最后自动执行 `verify_training_quality --version-prefix clean-risk`。
   - 支持 `-SkipCheck` 跳过轻量检查，支持 `-AllowQualityGateFail` 仅用于调试失败模型写库。

## 本次验证结果

- Python 静态编译通过。
- 本次没有自动执行正式训练，避免覆盖你当前库里的模型结果。
- 轻量 `--no-write` 验证显示，LightGBM 的风险识别已经从之前正式训练中的负提升恢复为正提升：
  - `lightgbm_walk_forward_direction_lift_over_baseline = +0.0397`
  - `lightgbm_walk_forward_down_precision = 0.6149`
  - `lightgbm_walk_forward_predicted_down_ratio = 0.1728`
  - 说明 DOWN 覆盖率已经从“大面积看空”收敛到更可用的风险筛选口径。
- 复测后确认，LSTM 仍然存在 walk-forward 负提升和风险覆盖率偏高的问题；生产集成逻辑已调整为：LSTM 继续训练和记录指标，但如果 `walk_forward` 负提升且风险覆盖率超过校准上限，就从最终 ensemble 权重中剔除，避免污染正式预测。
  - 复测中的 LSTM：`lstm_walk_forward_direction_lift_over_baseline = -0.1370`
  - 复测中的 LSTM：`lstm_walk_forward_predicted_down_ratio = 0.7505`
  - 最终集成权重：`ensemble_lightgbm_weight = 1.0`
- 特征/标签拆分后再次复测：
  - `collect metrics` 从约 44 秒降到约 14 秒。
  - 训练输出不再出现 `DataFrame is highly fragmented` 警告。
  - LightGBM 关键 walk-forward 指标保持稳定：`lift = +0.0397`，`down_precision = 0.6149`，`predicted_down_ratio = 0.1728`。

  - 控制台输出 `quality_gate=PASS`。

  - sklearn 的“预测了真实标签中不存在类别”警告已消失。

## 剩余限制

- 本次是数据质量和训练决策口径重构，不保证不重新训练就看到前端变化。
- 如果清洗后样本减少明显，需要重新补日线数据后再训练。
- 后续还可以继续增加 `data_quality_logs` 表，记录每条被丢弃数据的原因。
