# CI、文档结构与首页样式拆分说明

## 修改目的

本次修改解决三个工程维护问题：

1. GitHub 仓库缺少自动检查，代码推送后没有统一的质量证明。
2. `docs/` 根目录同时堆放历史修改说明、正式文档和答辩材料，查找成本高。
3. 首页 `index.html` 内联 CSS 较多，不利于后续维护和复用。

## 修改文件

- `.github/workflows/ci.yml`
- `frontend/index.html`
- `frontend/dashboard-home.css`
- `python/tests/test_stock_utils.py`
- `docs/README.md`
- `docs/changelog/`
- `docs/reports/`
- `docs/defense/`
- `docs/ppt/.gitkeep`
- `docs/changelog/代码修改说明_2026-05-30_CI文档结构与首页样式拆分.md`

## 具体修改

1. 新增 GitHub Actions 工作流：
   - 检查前端 JavaScript 语法。
   - 编译检查 Python 文件。
   - 运行 Python 单元测试。
   - 编译 Java 后端。
2. 将历史 `代码修改说明_*.md` 和机器学习优化方案移动到 `docs/changelog/`。
3. 将正式项目说明书移动到 `docs/reports/`。
4. 将答辩和功能展示材料移动到 `docs/defense/`。
5. 新增 `docs/README.md`，说明文档目录用途和维护规则。
6. 将首页内联样式拆分为 `frontend/dashboard-home.css`，`index.html` 只保留页面结构和脚本引用。
7. 更新 `test_stock_utils.py` 中的市场代码测试，和当前项目只支持 A 股 / 港股的工具函数口径保持一致。

## 剩余限制

- GitHub Actions 首次运行结果需要等推送后在 GitHub 页面查看。
- 首页样式已从 HTML 拆出，但子页面仍使用 `dash-pages.css`，后续可以继续统一设计变量。
