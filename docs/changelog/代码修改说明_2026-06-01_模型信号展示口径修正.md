# 代码修改说明：模型信号展示口径修正

## 修改目的

模型原始方向 `predicted_signal` 用于模型评估和 drift 检测，但它不应直接作为前端“可信看涨/看跌”展示依据。用户看到“看多 89 / 看空 2 / 观望 1”时，容易误以为系统强烈建议看涨；如果实盘随后大跌，会削弱系统可信度。

本次修改将模型页“信号分布”改为展示经过置信度过滤后的 `alert_signal`，让前端默认呈现保守告警信号。原始 `predicted_signal` 仍保留在数据库中，用于模型评估。

## 修改文件

- `java-backend/src/main/java/com/recruitment/backend/service/DashboardService.java`
- `frontend/dash-pages.js`
- `frontend/models.html`

## 具体改动

1. 后端 `signal_distribution` 从按 `predicted_signal` 聚合改为按 `alert_signal` 聚合。
2. 后端 `ml_predictions` 接口补充返回 `alert_signal` 字段，前端可同时区分原始模型信号和告警侧信号。
3. 前端 `renderSignalChart()` 优先读取 `alert_signal`，图表中心文案改为“保守信号”。
4. 模型页标题从“综合信号分布”改为“保守信号分布”，避免把原始模型方向包装成可直接依赖的交易信号。

## 剩余限制

- 该修改只修正展示口径，不会提高模型本身预测能力。
- LightGBM 仍是主要权重模型，前端和答辩时应说明模型信号是辅助风险参考，不能作为单独买卖依据。
