# 模型风险评分体系重构说明

## 修改目的

原来的模型展示主要围绕 UP / DOWN / WATCH 三分类，容易出现两个问题：LSTM 在部分训练结果中权重过高，把整体信号带偏；前端只能看到方向和置信度，不能解释“为什么认为有风险”。本次把模型部分改成“LightGBM 主导的下跌风险识别 + LSTM 序列辅助评分”。

## 修改文件

- `python/ml/stock_ml.py`
- `python/common/config.py`
- `python/.env.example`
- `python/sql/init.sql`
- `java-backend/src/main/java/com/recruitment/backend/service/DashboardService.java`
- `frontend/dashboard-home.js`
- `frontend/dash-pages.js`

## 核心改动

1. 生产融合权重重构：
   - 新增 `normalized_ensemble_weights()`。
   - 风险模式下 LightGBM 默认至少占 `0.85` 权重。
   - LSTM 默认最多占 `0.10` 权重；如果 walk-forward、下跌收益或 guard 结果不稳定，会继续降到 `0.05` 或 `0.03`。

2. 预测结果新增风险评分字段：
   - `model_risk_score`：模型下跌概率分数。
   - `technical_risk_score`：价格、量能、相对大盘/行业弱势证据分数。
   - `sequence_risk_score`：LSTM 序列辅助风险分数。
   - `final_risk_score`：最终下跌风险评分。

3. 数据库结构同步：
   - `ml_predictions` 和 `ml_prediction_history` 新增上述四个评分字段。
   - `python/sql/init.sql` 已同步更新，Java 启动时也会对旧库自动补字段。

4. 前端展示调整：
   - 首页预测表优先展示保守告警信号 `alert_signal`。
   - 模型页股票排行显示风险评分百分比。
   - 后端选股排行的风险分数加入 `final_risk_score`，高风险股票更容易进入风险榜。

```bat

set ML_LSTM_EPOCHS=5
```

## 剩余限制

- 这次解决的是“模型融合和展示口径”问题，不保证训练指标立即提升。
- 如果 LightGBM 本身下跌召回仍低，下一步应继续补充行业相对强弱、指数弱势和量价破位特征，而不是提高 LSTM 权重。
- 旧数据不会自动生成新增评分字段，需要重新训练写库后前端才会显示有效风险评分。
