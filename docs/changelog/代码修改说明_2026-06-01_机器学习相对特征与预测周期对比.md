# 代码修改说明：机器学习相对特征与预测周期对比

## 修改目的

本次修改解决两个问题：

1. 模型特征不能只依赖个股日线价格和成交量，需要补充更有金融解释价值的相对特征。
2. 训练目标不应固定为 3 日后方向，需要支持对比 1 日、3 日、5 日预测目标。

## 修改文件

- `python/common/config.py`
- `python/ml/stock_ml.py`
- `python/ml/train_daily_predict.py`
- `python/ml/tune_daily_params.py`
- `python/ml/model_drift.py`

## 具体改动

### 1. 增加相对行情特征

在 `stock_ml.py` 的特征工程中补充以下字段：

- `relative_sector_change_pct`：个股当前涨跌幅相对所属行业平均涨跌幅。
- `relative_market_change_pct`：个股当前涨跌幅相对同市场平均涨跌幅。
- `current_vs_market_index_return`：个股当前涨跌幅相对对应大盘指数 1 日涨跌幅。
- `relative_market_index_return_1` / `relative_market_index_return_3` / `relative_market_index_return_5`：个股 1/3/5 日收益相对对应市场指数收益。
- `relative_hs300_return_5`：个股 5 日收益相对沪深 300。
- `volume_ratio_20_raw`：成交量 / 近 20 日均量。
- `price_vs_ma20`：价格距离 20 日均线比例。
- `up_streak_3` / `down_streak_3`：近 3 个交易点连续上涨或下跌强度。
- `volume_spike_20`：是否达到 20 日均量 2 倍。
- `volume_up_breakout` / `volume_down_breakout`：是否放量上涨或放量下跌。

### 2. 支持可配置预测周期

新增配置：

- `ML_PREDICTION_HORIZON`，默认 `3`。
- `ML_HORIZON_EXPERIMENTS`，默认 `1,3,5`。

训练入口新增参数：

```powershell
--prediction-horizon 1
--horizon-experiments 1,3,5
```

`prepare_dataset()` 现在支持 `prediction_horizon`，会按所选周期生成 `next_price`、`next_return`、`next_direction`。

### 3. 增加周期对比指标

新增 `horizon_experiment_metrics()`，用于输出不同预测周期的标签分布：

- 样本数
- 股票数
- DOWN / UP / WATCH 占比
- majority baseline accuracy

这些指标只用于诊断，不参与模型选择。

### 4. Drift 检测跟随训练周期

`model_drift.py` 的默认 horizon 改为读取 `ML_PREDICTION_HORIZON`。

正式训练后执行 drift 检测时，会使用本次训练命令中的 `--prediction-horizon`，避免训练目标和 drift 评估目标不一致。

### 5. 修复轻量单模型验证路径

修复了只训练单个模型时 `collect_metrics()` 中 ensemble 权重变量未初始化的问题。现在可以稳定运行：

```powershell
--models lightgbm --no-write
```

## 验证命令与结果

### 静态检查

```powershell
D:\anaconda3\envs\MachineLearn\python.exe -m py_compile python\common\config.py python\ml\stock_ml.py python\ml\train_daily_predict.py python\ml\tune_daily_params.py python\ml\model_drift.py
```

结果：通过。

### 轻量训练验证

```powershell
$env:ML_MAX_TRAIN_ROWS='12000'
$env:ML_LSTM_EPOCHS='2'
$env:ML_LSTM_MAX_SEQUENCES='12000'
D:\anaconda3\envs\MachineLearn\python.exe -m python.ml.train_daily_predict --version-label horizon-check --models lightgbm --prediction-horizon 1 --horizon-experiments 1,3,5 --allow-stale-data --no-write
```

结果：通过。训练、预测、指标收集均完成，未写库。

本次轻量验证输出了 1/3/5 日目标的标签分布。例如在 `0.015` 方向阈值下：

- T+1：WATCH 占比约 `60.57%`
- T+3：WATCH 占比约 `37.92%`
- T+5：WATCH 占比约 `29.68%`

这说明 1 日目标在当前阈值下更容易退化成 WATCH 多数类，后续正式调参应重点比较 T+3 和 T+5 的真实 walk-forward 表现。

## 建议正式训练命令

先按当前生产组合 LightGBM + LSTM 训练 T+3：

```powershell
$env:ML_MAX_TRAIN_ROWS='120000'
$env:ML_LSTM_EPOCHS='60'
$env:ML_LSTM_MAX_SEQUENCES='80000'
D:\anaconda3\envs\MachineLearn\python.exe -m python.ml.train_daily_predict --version-label daily-best --models lightgbm,lstm --prediction-horizon 3 --horizon-experiments 1,3,5
```

如需测试 5 日目标：

```powershell
$env:ML_MAX_TRAIN_ROWS='120000'
$env:ML_LSTM_EPOCHS='60'
$env:ML_LSTM_MAX_SEQUENCES='80000'
D:\anaconda3\envs\MachineLearn\python.exe -m python.ml.train_daily_predict --version-label daily-h5 --models lightgbm,lstm --prediction-horizon 5 --horizon-experiments 1,3,5
```

## 剩余限制

- 本次只加入特征和目标周期支持，没有自动证明新特征一定提升模型效果。
- 轻量验证只训练了 LightGBM，完整结论仍要以 LightGBM + LSTM 正式训练结果为准。
- T+1 在当前阈值下 WATCH 占比偏高，可能不适合作为最终答辩口径。
