# 代码修改说明：训练写库与漂移检测修复

## 修改目的

本次按正式训练流程运行 LightGBM + LSTM 后，发现并修复 3 个问题：

1. ensemble 预测阶段 `weighted_predictions` 被损坏注释吞掉，导致训练完成后无法生成最终预测。
2. `ml_prediction_history` 建表 SQL 中中文 `COMMENT` 字符串被编码损坏，导致 MySQL 写库失败。
3. drift 检测把日线模型拿实时 tick 做 T+3 评估，口径不一致。

同时修复了 `alert_signal` 可能写成字符串 `nan` 的问题。

## 修改文件

- `python/ml/stock_ml.py`
- `python/ml/model_drift.py`
- `python/ml/train_daily_predict.py`
- `docs/changelog/代码修改说明_2026-06-01_训练写库与漂移检测修复.md`

## 具体改动

### 1. 修复 ensemble 预测初始化

`predict_latest_ensemble()` 中恢复：

```python
weighted_predictions: list[tuple[str, float, pd.DataFrame]] = []
```

否则 LightGBM + LSTM 都训练完成后，会在预测阶段报：

```text
NameError: name 'weighted_predictions' is not defined
```

### 2. 修复写库建表 SQL

`ml_prediction_history` 的建表 SQL 删除损坏的中文 `COMMENT`，避免 MySQL 解析失败。

失败原因示例：

```text
pymysql.err.ProgrammingError: (1064, "You have an error in your SQL syntax ...")
```

### 3. 修复 drift 评估口径

`model_drift.py` 新增：

```text
--model-version
--price-source daily|auto
```

默认 `price-source=daily`。日线模型只用 `daily_stock_bars` 做 T+N 评估，不再自动 fallback 到 `price_ticks`。

训练入口现在调用：

```python
check_model_drift(..., model_version=f"ensemble-{version}", price_source="daily")
```

这样刚训练出的模型如果还没有未来 3 个交易日数据，会返回 `ready=False`，不会误用实时 tick 得出漂移结论。

### 4. 修复 alert_signal 的 nan

`apply_alert_confidence_filter()` 和 `write_results()` 都增加兜底：

- `predicted_signal` 只允许 `UP / DOWN / WATCH`
- `alert_signal` 只允许 `UP / DOWN / WATCH`
- 非法值、空值、NaN 统一写成 `WATCH`

## 剩余限制

- LightGBM 已经略高于 majority baseline，但提升幅度仍小。
- LSTM 单次验证集表现有提升，但 walk-forward 仍不稳定。
- 当前最可信的展示口径应是：系统完成训练和预测，模型指标略高于基线，但仍属于谨慎参考，不应宣传为高精度预测模型。
