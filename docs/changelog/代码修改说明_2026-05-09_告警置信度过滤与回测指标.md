# 代码修改说明：告警置信度过滤与回测指标

## 1. 修改目标

本次修改的目标是让“股票价格预测与告警”更贴近业务使用场景：

- 对低置信度的 `UP` / `DOWN` 方向告警进行降级，避免模型在把握不足时仍然触发方向性告警。
- 在训练结果中增加告警回测指标，不只看 `accuracy`、`balanced_accuracy`，还要看方向告警覆盖率、命中率和告警后的平均收益表现。

## 2. 修改文件

- `python/.env`
- `python/common/config.py`
- `python/ml/stock_ml.py`

## 3. 具体修改

### 3.1 新增告警置信度阈值配置

在 `python/.env` 中新增：

```env
ML_ALERT_CONFIDENCE_THRESHOLD=0.60
```

`python/common/config.py` 中通过 `settings.ml_alert_confidence_threshold` 读取该配置。当前默认值为 `0.60`。

含义是：当模型输出 `UP` 或 `DOWN`，但置信度低于 `0.60` 时，最终写入数据库和前端展示的信号会降级为 `WATCH`。

### 3.2 新增低置信度告警过滤

在 `python/ml/stock_ml.py` 中新增 `apply_alert_confidence_filter()`：

- 只处理 `UP` / `DOWN`。
- 不改变原始概率列。
- 当 `confidence < ML_ALERT_CONFIDENCE_THRESHOLD` 时，将 `predicted_signal` 改为 `WATCH`。
- 如果存在 `predicted_direction`，同步改为 `DIRECTION_FLAT`。

该过滤已经接入：

- `predict_latest_tabular()`
- `predict_latest_lstm()`
- `predict_latest_ensemble()`

因此单模型预测和 ensemble 预测都会应用同一套告警策略。

### 3.3 新增告警回测指标

在 `python/ml/stock_ml.py` 中新增 `alert_backtest_metrics()`，训练时会输出以下类型的指标：

- `alert_backtest_raw_coverage`：模型原始 UP/DOWN 覆盖率。
- `alert_backtest_coverage`：经过置信度过滤后的 UP/DOWN 覆盖率。
- `alert_backtest_filtered_ratio`：被降级为 WATCH 的比例。
- `alert_backtest_watch_ratio`：最终 WATCH 比例。
- `alert_backtest_hit_rate`：UP/DOWN 告警整体命中率。
- `alert_backtest_up_hit_rate`：UP 告警命中率。
- `alert_backtest_down_hit_rate`：DOWN 告警命中率。
- `alert_backtest_up_avg_return`：UP 告警后的平均真实收益。
- `alert_backtest_down_avg_inverse_return`：DOWN 告警后的平均反向收益。

这些指标已经接入：

- RandomForest
- LightGBM
- LSTM

## 4. 解决的问题

修改前的问题是：模型的三分类准确率不高时，前端仍可能展示很多低把握的 `UP` / `DOWN`，容易让系统看起来“乱报方向”。

修改后：

- 模型仍然可以学习 UP / DOWN / WATCH 三分类。
- 低置信度方向预测不会直接变成方向告警。
- 训练输出能看到告警是否真的有业务价值，而不是只看分类准确率。

## 6. 后续训练命令

本次没有执行训练。你可以在本地运行：

```powershell
python -m python.ml.train_daily_predict
```

如果想临时调整告警置信度，例如改成 `0.65`，可以先在 `python/.env` 中修改：

```env
ML_ALERT_CONFIDENCE_THRESHOLD=0.65
```

然后重新训练并观察 `alert_backtest_*` 指标。

## 7. 剩余问题

- 置信度阈值 `0.60` 是保守初始值，需要结合训练输出中的 `alert_backtest_hit_rate` 和 `alert_backtest_coverage` 再调。
- 如果阈值过高，UP/DOWN 会大量变成 WATCH，告警更稳但覆盖率会下降。
- 如果阈值过低，覆盖率会提高，但方向误报也可能增加。
