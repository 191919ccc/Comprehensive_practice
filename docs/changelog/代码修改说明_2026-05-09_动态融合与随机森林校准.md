# 代码修改说明：动态融合权重与随机森林概率校准

## 1. 修改目标

上一轮训练中，模型实际表现和固定融合权重不完全匹配：

- LSTM 的 walk-forward 表现更稳。
- LightGBM 在 walk-forward lift 上低于基线，但原来固定占 60%。
- RandomForest 没有参与 ensemble，浪费了树模型和时序模型之间的多样性。
- RandomForest 的概率未经校准，直接平均概率不够公平。

本次修改目标是让 ensemble 更贴近真实验证表现。

## 2. 修改文件

- `python/ml/stock_ml.py`

## 3. 具体修改

### 3.1 RandomForest 方向模型加入概率校准

`train_random_forest()` 中的方向分类器改为：

```python
CalibratedClassifierCV(
    estimator=RandomForestClassifier(...),
    method="isotonic",
    cv=3,
)
```

这样 RandomForest 输出的 `predict_proba()` 会经过等温回归校准，再参与后续概率融合。

### 3.2 Ensemble 从固定权重改为动态权重

原来：

```text
LightGBM = 0.6
LSTM = 0.4
```

现在根据每个模型的：

```text
walk_forward_balanced_direction_accuracy
```

动态计算权重。

RandomForest 参与融合，但做 0.5 折扣：

```python
if result.name == "random_forest":
    score *= 0.5
```

这样 RF 能提供多样性，但不会因为验证集偶然表现过高而压过 LSTM / LightGBM。

### 3.3 保存训练指标，保证 predict_daily 也能动态融合

原来训练后保存模型文件，但重新加载模型时 `metrics={}`，导致 `predict_daily` 无法知道训练时的 walk-forward 表现。

现在 `save_models()` 会额外保存：

```text
model-metrics-{version}.joblib
```

`load_latest_model_results()` 会读取这个指标文件，并恢复到每个 `ModelResult.metrics` 中。

这样以后执行：

```powershell
python -m python.ml.predict_daily
```

也能使用训练时的动态 ensemble 权重。

### 3.4 模型评分公式重调

`direction_quality_score()` 已调整为 walk-forward 优先：

```text
walk_forward_balanced_accuracy  0.35
walk_forward_macro_f1           0.25
main balanced_accuracy          0.20
main macro_f1                   0.10
lift                            0.05
return_mae                     -0.05
```

同时保留之前加入的：

- DOWN 告警错边惩罚
- UP 告警收益奖励

因此模型选择既看稳定分类表现，也看告警业务质量。

### 3.5 Ensemble 权重写入指标表

`collect_metrics()` 现在会额外写入：

- `ensemble / lstm_weight`
- `ensemble / lightgbm_weight`
- `ensemble / random_forest_weight`

训练后可以直接在 `ml_model_metrics` 中查看实际融合权重。

## 6. 注意事项

- RandomForest 加校准后训练会比原来慢一些。
- 新的 ensemble 权重只有从本次训练开始才会保存到 `model-metrics-{version}.joblib`。
- 旧模型如果没有指标文件，`predict_daily` 会使用默认权重回退。
