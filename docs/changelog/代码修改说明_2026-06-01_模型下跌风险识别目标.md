# 模型下跌风险识别目标修改说明

## 修改目的

原模型主目标是 `UP / DOWN / WATCH` 三分类方向预测，但项目定位是股票实时风险告警系统。三分类准确率容易被 `WATCH` 或看多信号稀释，不能直接回答“是否提前识别下跌风险”。本次把训练目标改为可配置的下跌风险识别模式，重点优化：

- 下跌召回率；
- 看空覆盖率；
- 大跌样本识别能力；
- 下跌告警命中率。

## 修改文件

- `python/common/config.py`
- `python/.env.example`
- `python/.env`（本地忽略文件，用于当前机器训练）
- `python/ml/stock_ml.py`
- `python/ml/train_daily_predict.py`

## 关键代码变化

1. 新增 `ML_TARGET_MODE`
   - 默认值：`downside_risk`
   - 可选值：
     - `downside_risk`：训练未来 N 日下跌风险；
     - `direction`：恢复旧的 `UP / DOWN / WATCH` 三分类方向预测。

2. 新增风险标签
   - 未来 N 日最终收益跌破阈值；
   - 或未来 N 日最低价回撤跌破阈值；
   - 或未来 N 日进入高波动区且出现明显回撤。

3. 风险模式下预测信号只输出 `DOWN / WATCH`
   - 不再把正收益预测强行覆盖成 `UP`；
   - 避免模型页继续出现大量“看多”但项目实际关注风险的口径冲突。

4. 调参和集成权重偏向下跌风险
   - 决策阈值搜索优先考虑 `down_recall`、`down_precision`、`predicted_down_ratio`；
   - 集成权重优先奖励下跌召回、下跌精度、下跌告警命中率和下跌后的反向收益。

5. 保留旧逻辑
   - 如果本次效果不好，把 `ML_TARGET_MODE` 改为 `direction` 后重新训练即可恢复旧目标。

## 建议训练命令

先跑轻量验证，不写库：

```powershell
set ML_TARGET_MODE=downside_risk
set ML_MAX_TRAIN_ROWS=15000
set ML_LSTM_EPOCHS=5
set ML_LSTM_MAX_SEQUENCES=30000
D:\anaconda3\envs\MachineLearn\python.exe -m python.ml.train_daily_predict --version-label risk-check --models lightgbm,lstm --prediction-horizon 5 --horizon-experiments 1,3,5 --no-write
```

重点看这些指标是否改善：

- `down_recall`
- `predicted_down_ratio`
- `alert_backtest_down_count`
- `alert_backtest_down_hit_rate`
- `alert_backtest_down_avg_inverse_return`
- `target_downside_risk_enabled = 1.0`

如果轻量验证效果可以，再正式写库：


## 回退方式

不改代码，只恢复旧训练目标：

```powershell
set ML_TARGET_MODE=direction
D:\anaconda3\envs\MachineLearn\python.exe -m python.ml.train_daily_predict --version-label daily-direction --models lightgbm,lstm --prediction-horizon 5 --horizon-experiments 1,3,5
```

如果要撤销本次代码改动：

```powershell
git restore -- python/common/config.py python/.env.example python/ml/stock_ml.py python/ml/train_daily_predict.py docs/changelog/代码修改说明_2026-06-01_模型下跌风险识别目标.md
```

本地 `python/.env` 是忽略文件，如需恢复旧模式，手动把 `ML_TARGET_MODE=downside_risk` 改为 `ML_TARGET_MODE=direction`，或删除 `ML_TARGET_MODE` 和 `ML_RISK_VOLATILITY_THRESHOLD` 两行。

## 剩余限制

- 本次是训练目标和决策口径修改，最终效果必须重新训练后才能判断。
- 风险模式会减少甚至取消 `UP` 信号，这是有意设计，目的是让模型服务风险告警，而不是做多空方向预测。
- 如果训练后 `DOWN` 覆盖率过高，需要继续调低风险标签敏感度或提高 `ML_RISK_VOLATILITY_THRESHOLD`。
