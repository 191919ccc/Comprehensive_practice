# LightGBM 与 LSTM 最优参数回写说明

## 修改目标

根据快速分组测试结果，把当前效果最好的 LightGBM 组内参数和全局最优 LSTM 参数回写到正式训练配置中，后续执行 `train_daily_predict` 时直接使用这组参数。

## 测试依据

测试命令：

```powershell
python -m python.ml.tune_daily_params --models lightgbm,lstm --max-rows 30000 --lstm-max-sequences 12000 --lstm-epochs 10 --lstm-patience 4
```

测试结果摘要：

- LightGBM 最优组：`lgbm_more_leaves`
  - `score=0.3630`
  - `walk_forward_balanced_direction_accuracy=0.4062`
  - `walk_forward_direction_lift_over_baseline=-0.0243`
- LSTM 最优组：`lstm_wider`
  - `score=0.3733`
  - `walk_forward_balanced_direction_accuracy=0.4108`
  - `walk_forward_direction_lift_over_baseline=0.0449`

## 具体修改

1. `python/ml/stock_ml.py`
   - LightGBM 参数从当前默认组切换为 `lgbm_more_leaves`：
     - `n_estimators=350`
     - `num_leaves=63`
     - `min_child_samples=40`
     - `subsample=0.8`
     - `colsample_bytree=0.85`
     - `reg_lambda=1.5`

2. `python/common/config.py`
   - LSTM 默认参数切换为 `lstm_wider`：
     - `ML_LSTM_HIDDEN_SIZE=64`
     - `ML_LSTM_DROPOUT=0.35`
     - `ML_LSTM_LEARNING_RATE=0.0005`
     - `ML_LSTM_WEIGHT_DECAY=0.0015`

3. 文档同步
   - 在参数分组测试说明中补充本轮测试结果、最佳组和回写后的训练命令。

- LightGBM 虽然采用了组内最佳参数，但本轮快速测试里仍低于多数类基线，不能单独作为强模型宣传。
- LSTM 的 `lstm_wider` 在快速测试中相对基线为正，是本轮更值得重点观察的参数组。
