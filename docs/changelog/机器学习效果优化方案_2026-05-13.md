# 机器学习效果优化方案

## 1. 当前问题判断

当前模型效果偏弱，不是单纯“参数没调好”的问题。最近一次正式训练和快速分组测试暴露出几个核心问题：

- LightGBM 虽然在组内有最优参数，但 `walk_forward_direction_lift_over_baseline` 仍为负，说明它没有稳定超过多数类基线。
- LSTM 的 `lstm_wider` 参数组相对基线为正，但提升幅度还不大，需要继续验证是否稳定。
- 当前页面展示的准确率容易误导，真正应该优先看 `walk_forward_balanced_direction_accuracy`、`macro_f1` 和 `lift_over_baseline`。
- 模型预测的是 T+3 方向，股票短期方向本身噪声很高，仅靠单股历史价格和技术指标很难稳定强预测。

结论：后续优化必须先提高数据质量和标签质量，再做模型和参数优化。否则继续盲目调参，只会在小样本验证集上抖动。

## 2. 优化目标

短期目标：

- 让最佳模型的 `walk_forward_direction_lift_over_baseline` 稳定为正。
- 让 `walk_forward_balanced_direction_accuracy` 稳定高于多数类基线对应水平。
- 降低模型在 UP、DOWN、WATCH 三类之间的偏置。

中期目标：

- 形成可解释的“辅助判断模型”，而不是宣传为高准确率预测器。
- 页面展示指标从“单一准确率”升级为“模型是否超过基线”的表达。

长期目标：

- 建立定期数据扩充、重训、漂移检测、回测验证流程。

## 3. 第一阶段：数据层优化

### 3.1 扩充股票池

当前已有 92 只股票，相比之前有明显扩充，但还不够覆盖市场结构。建议继续扩到 150 到 300 只，覆盖：

- 上证主板
- 深证主板
- 创业板
- 科创板代表股
- 港股代表股
- 金融、消费、医药、科技、能源、周期、地产、汽车等行业

原因：模型现在学习到的行业轮动和市场相对强弱特征还不稳定，股票池越窄，越容易学到局部噪声。

### 3.2 延长历史周期

建议日线历史至少覆盖 3 到 5 年。

重点覆盖：

- 上涨阶段
- 下跌阶段
- 横盘阶段
- 高波动阶段
- 低波动阶段

如果只覆盖最近一两年，模型容易只适应某一种市场状态。

### 3.3 数据质量清洗

需要增加数据检查脚本，检查：

- 每只股票样本数是否过少。
- 是否存在连续重复价格。
- 是否存在成交量为 0 或异常大。
- 是否存在停牌导致的长时间缺口。
- `open/high/low/close` 是否逻辑异常，例如 `high < low`、`close` 超出高低价范围。

建议新增一个数据体检输出：

```text
reports/ml_data_quality/daily_data_quality_*.csv
```

每次训练前先看数据质量，再决定是否训练。

## 4. 第二阶段：标签层优化

### 4.1 重新评估 T+3 标签

当前预测目标是 T+3，即 3 个交易日后的方向。这个目标可能过短，噪声很大。

建议分组测试：

- T+1
- T+3
- T+5
- T+10

判断标准不是普通准确率，而是：

- `walk_forward_balanced_direction_accuracy`
- `walk_forward_direction_macro_f1`
- `walk_forward_direction_lift_over_baseline`
- UP/DOWN 信号的实际平均收益

如果 T+5 或 T+10 更稳定，应考虑把主模型改成更长预测周期。

### 4.2 重新评估方向阈值

当前日线方向阈值是 `0.015`，也就是涨跌幅在正负 1.5% 内归为 WATCH。

建议继续测试：

- `0.005`
- `0.010`
- `0.015`
- `0.020`
- `0.030`

如果 WATCH 类占比过高，模型容易学成“多数类判断”；如果阈值过低，UP/DOWN 会被噪声污染。

### 4.3 引入收益风险标签

只预测涨跌方向不够，可以增加辅助标签：

- 未来 N 日最大回撤
- 未来 N 日最大上涨
- 风险收益比
- 是否触发止损区间

这样模型不只判断方向，还能判断“这个信号值不值得关注”。

## 5. 第三阶段：特征层优化

### 5.1 增加市场状态特征

当前已有指数相对收益、行业相对收益等特征，但还可以补充：

- 大盘 20 日趋势状态
- 大盘波动率分位数
- 行业 5 日/20 日强弱排名
- 个股相对行业强弱
- 个股相对沪深 300 强弱

目标：让模型知道“当前是市场普涨、普跌、轮动、还是震荡”。

### 5.2 增加成交量结构特征

建议新增：

- 量价背离
- 放量上涨
- 放量下跌
- 缩量上涨
- 缩量下跌
- 近 5 日成交量分位数
- 近 20 日成交量分位数

原因：短期方向对成交量状态很敏感，只看价格会漏掉资金行为。

### 5.3 增加波动率和位置特征

建议新增：

- 近 20 日价格位置
- 近 60 日价格位置
- ATR 类波动指标
- 布林带位置
- 距离 20 日高点/低点比例

这些特征比单纯均线差值更能表达“当前价格处于高位还是低位”。

## 6. 第四阶段：模型层优化

### 6.1 LightGBM

当前已回写的 LightGBM 组是：

```text
n_estimators=350
learning_rate=0.02
num_leaves=63
min_child_samples=40
subsample=0.8
colsample_bytree=0.85
reg_lambda=1.5
```

下一步建议继续扩展分组：

- 更强正则：`reg_lambda=3.0/5.0`
- 更大 `min_child_samples=80/120`
- 更低学习率：`learning_rate=0.01`
- 更浅树：`num_leaves=15/31`

LightGBM 当前最大问题是 walk-forward 低于基线，所以重点不是追求训练集更高，而是防过拟合。

### 6.2 LSTM

当前最优快速组是：

```text
window=30
hidden=64
dropout=0.35
learning_rate=0.0005
weight_decay=0.0015
```

下一步建议测试：

- `window=20/30/45`
- `hidden=64/96`
- `dropout=0.35/0.45`
- `learning_rate=0.0003/0.0005`
- `weight_decay=0.0015/0.003`

如果 LSTM 在更长正式训练中仍能保持正 lift，就把它作为主方向模型；LightGBM 作为辅助特征模型。

### 6.3 集成策略

当前 ensemble 已动态加权，但后续应加入硬规则：

- walk-forward lift 为负的模型，权重强制降到很低。
- 最近漂移检测失败的模型，权重降低。
- 只有超过基线的模型才能成为最佳模型。

这样可以避免“看似准确率高，但实际不如多数类基线”的模型参与过多。

## 7. 第五阶段：验证和回测

### 7.1 固定验证口径

以后判断模型好坏只看这几个指标：

```text
walk_forward_balanced_direction_accuracy
walk_forward_direction_macro_f1
walk_forward_direction_lift_over_baseline
alert_backtest_hit_rate
alert_backtest_up_avg_return
alert_backtest_down_avg_inverse_return
```

不再用单一 `direction_accuracy` 做主要结论。

### 7.2 增加分市场验证

必须分开看：

- SH
- SZ
- 创业板
- HK
- 各行业

如果模型只在某些行业有效，页面就应该只对这些行业给强信号。

### 7.3 增加时间段验证

按时间切分：

- 最近 3 个月
- 最近 6 个月
- 最近 12 个月
- 全部历史

如果全历史好、最近 3 个月差，说明模型已经漂移。

## 8. 页面展示优化

页面上不要只显示一个“40.2%”。建议改成：

```text
模型验证：偏弱 / 可观察 / 有效
平衡准确率：xx%
多数类基线：xx%
相对基线：+x.x / -x.x 个百分点
主模型：LSTM / LightGBM / Ensemble
```

判断规则：

- lift < 0：显示“低于基线，谨慎参考”
- lift 0 到 0.03：显示“略高于基线，仅作辅助”
- lift > 0.03：显示“高于基线，可观察”

这样答辩时更诚实，也更专业。

## 10. 最重要的结论

这个项目的机器学习要改善，优先级应该是：

```text
数据覆盖 > 标签定义 > 验证口径 > 模型参数 > 页面展示
```

当前不是“换一个模型就能明显变好”的阶段。下一步最关键的是验证已回写的 LSTM 参数在正式训练中是否稳定超过基线；如果不能，就必须转向扩充数据和重做标签，而不是继续盲目调参。
