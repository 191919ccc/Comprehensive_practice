# 机器学习优化方案落地说明

## 修改目标

根据 `docs/机器学习效果优化方案_2026-05-13.md`，先落地能直接改善训练流程和结果判断的关键改动：

- 训练前增加数据质量体检。
- 增加预测周期和方向阈值的标签测试工具。
- 收紧最佳模型选择和集成权重，避免低于多数类基线的模型被高估。
- 前端模型卡片显示相对多数类基线，避免只看单一准确率。

## 修改文件

- `python/ml/daily_data_quality.py`
- `python/ml/tune_label_targets.py`
- `python/ml/stock_ml.py`
- `frontend/index.html`
- `docs/代码修改说明_2026-05-14_机器学习优化方案落地.md`

## 具体修改

### 1. 新增数据质量体检

新增 `python/ml/daily_data_quality.py`。

作用：

- 检查每只股票样本数。
- 检查最大日期缺口。
- 检查重复收盘价比例。
- 检查成交量为 0 的比例。
- 检查 OHLC 逻辑异常，例如 `high < low` 或收盘价超出高低价范围。
- 输出 symbol 级报告和 summary 报告。

运行命令：

```powershell
cd E:\作业\ww\Comprehensive_practice
python -m python.ml.daily_data_quality
```

输出目录：

```text
reports/ml_data_quality/
```

### 2. 新增标签目标测试

新增 `python/ml/tune_label_targets.py`。

作用：

- 批量测试不同预测周期：
  - T+1
  - T+3
  - T+5
  - T+10
- 批量测试不同方向阈值：
  - 0.005
  - 0.010
  - 0.015
  - 0.020
  - 0.030
- 输出每组标签的 UP/DOWN/WATCH 占比、多数类基线、收益波动等信息。

运行命令：

```powershell
cd E:\作业\ww\Comprehensive_practice
python -m python.ml.tune_label_targets --horizons 1,3,5,10 --thresholds 0.005,0.01,0.015,0.02,0.03
```

输出目录：

```text
reports/ml_label_targets/
```

### 3. 收紧模型选择

修改 `python/ml/stock_ml.py`：

- `select_best_model()` 现在优先选择 `walk_forward_direction_lift_over_baseline > 0` 的模型。
- 如果所有模型都低于基线，才退回到原来的综合评分选择。
- 避免低于多数类基线的模型被误判为最佳模型。

### 4. 收紧集成权重

修改 `ensemble_weight_score()`：

- 如果模型 `walk_forward_direction_lift_over_baseline <= -0.03`，权重直接压到极低。
- 避免明显低于基线的模型在 ensemble 中占高权重。

### 5. 优化前端指标展示

修改 `frontend/index.html`：

- `walk_forward_balanced_direction_accuracy` 显示为 `walk-forward balanced`。
- 如果当前显示的是 walk-forward 指标，基线同步使用 `walk_forward_majority_baseline_accuracy`。
- 模型卡片显示：

```text
基线 xx% · 相对 +x.xpt / -x.xpt
```

- 如果相对基线为负，模型状态显示：

```text
低于基线，谨慎参考
```

## 推荐执行顺序

1. 先做数据体检：

```powershell
python -m python.ml.daily_data_quality
```

2. 再做标签目标测试：

```powershell
python -m python.ml.tune_label_targets --horizons 1,3,5,10 --thresholds 0.005,0.01,0.015,0.02,0.03
```

3. 根据标签测试结果决定是否继续使用 T+3、0.015 阈值。

4. 再正式训练：

```powershell
python -m python.ml.train_daily_predict
```

## 仍需观察的问题

- 这些改动会让训练流程更可靠，但不会直接保证模型立刻大幅提升。
- 如果标签测试显示 T+3/0.015 本身多数类基线过高或 WATCH 占比异常，应优先改预测周期或方向阈值。
- 如果数据体检发现大量缺口、重复价或 OHLC 异常，应先修数据再训练。
