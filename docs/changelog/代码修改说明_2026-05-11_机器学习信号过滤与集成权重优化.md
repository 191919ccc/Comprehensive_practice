# 机器学习信号过滤与集成权重优化说明

## 修改目标

本次修改针对 2026-05-11 日线模型训练结果中暴露的问题：LightGBM 日志过多、LSTM 在集成中权重偏高、DOWN 信号样本过少且容易失真、模型整体仍低于多数类基线。目标是让输出信号更保守、更适合答辩说明，并减少无效强信号。

## 修改文件

- `python/ml/stock_ml.py`
- `python/.env`
- `python/.env.example`

## 具体修改

1. LightGBM 训练参数改为静默模式：
   - `verbose=-1`
   - `verbosity=-1`
   - `force_col_wise=True`

   这样训练时不再输出大量 `Trained a tree...` 调试日志，同时消除 LightGBM 自动测试列/行并行方式的提示。

2. 预测结果增加 `predicted_return`：
   - 表格模型预测时保存收益率预测。
   - LSTM 预测时保存收益率预测。
   - 集成模型根据 `predicted_next_price` 和 `last_price` 反推集成收益率。

   这样后续信号过滤不只看分类概率，也能结合价格收益方向。

3. 强化 UP/DOWN 信号过滤：
   - UP 置信度阈值从 `0.70` 调整为 `0.72`。
   - DOWN 置信度阈值从 `0.75` 调整为 `0.85`。
   - DOWN 信号必须同时满足预测收益率低于负方向阈值，否则降级为 `WATCH`。
   - 训练回测指标同步使用这套过滤规则，避免回测口径和实际写库口径不一致。

   这会减少弱 DOWN 和低置信 UP 被写入结果表。

4. 调整集成权重计算：
   - 不再只依赖 walk-forward balanced accuracy。
   - 加入 balanced accuracy、macro F1、相对基线 lift、告警命中率、告警覆盖率。
   - 对低于基线的模型降权。
   - 对 LSTM 做额外降权，避免 LSTM 在当前验证表现偏弱时仍占接近 40% 权重。
   - 对没有告警覆盖的模型降权。

5. 示例配置同步：
   - `ML_TRAINING_LOOKBACK_DAYS=0`，默认使用全部历史数据。
   - 增加 UP/DOWN 告警阈值配置示例。

## 剩余问题

- 本次修改会让信号更保守，但不会凭空提升模型真实预测能力。
- 上一轮训练的 drift 检查结果为 `drift=True`，后续还应考虑在写库前接入 drift 状态，将强信号整体降级或在前端明确展示“漂移中”。
- 如果重新训练后指标仍低于基线，下一步应优先做特征和标签质量检查，而不是继续加模型复杂度。
