# LightGBM 与前端行情显示修正说明

日期：2026-05-18

## 修改目标

1. 修复 LightGBM 训练时出现的特征维度不一致问题，例如 `X has 166 features, but LGBMClassifier is expecting 164 features as input`。
2. 清理前端首页中残留的美股相关命名和展示痕迹。
3. 修正实时行情图中开盘价口径不稳定、日线预测图 tooltip 中开盘价和收盘价重复显示的问题。

## 修改文件

- `python/ml/stock_ml.py`
- `frontend/index.html`

## 具体修改

### 1. LightGBM 特征维度固定

在 `python/ml/stock_ml.py` 中新增 `CATEGORICAL_FEATURES` 和 `tabular_category_values()`：

- 将 `symbol`、`category`、`sector` 统一作为类别特征管理。
- 在同一次训练数据集上提前收集完整类别集合。
- `build_preprocessor()` 支持传入固定类别集合，并交给 `OneHotEncoder(categories=...)`。
- `train_tabular_models()` 的价格模型、方向模型、校准模型共用同一套类别维度。
- `walk_forward_direction_metrics()` 的每个 forward fold 也共用同一套类别维度。

这样可以避免不同切片中股票代码、分类、行业数量不同，导致 LightGBM 看到的 one-hot 输入维度变化。

### 2. 前端美股残留清理

在 `frontend/index.html` 中：

- 将旧的 `usIndex` 改为 `stockCountKpi`。
- 将旧的 `usDelta` 改为 `stockSourceMeta`。
- 将展示文案改为 `SH/SZ/HK · n 个数据源`，不再使用英文 `sources`。

### 3. 开盘价和收盘价显示修正

在 `frontend/index.html` 中：

- 图例从 `实际价格` 调整为 `收盘/最新价`，让实时和日线两种模式都能解释清楚。
- `chartTooltipOptions()` 增加：
  - `fallbackOpen`
  - `preferFallbackOpen`
  - `showOpenCloseExtra`
- 实时分钟图：
  - 开盘价优先使用当前主股票的 `main.open_price`。
  - 收盘/最新价使用当前分钟点的 `last_price`。
- 日线预测图：
  - 日线图本身已经有 `日线开盘价` 和 `日线收盘价` 两条曲线。
  - tooltip 不再额外追加开盘价、收盘价，避免同一个数值显示两次。

## 已验证

执行过以下检查：

```powershell
python -m py_compile .\python\ml\stock_ml.py
node -e "const fs=require('fs'); const html=fs.readFileSync('frontend/index.html','utf8'); const script=[...html.matchAll(/<script[^>]*>([\s\S]*?)<\/script>/g)].map(m=>m[1]).join('\n'); new Function(script); console.log('frontend script syntax ok');"
Get-ChildItem -Path .\frontend -File -Include *.html,*.js,*.css | ForEach-Object { Select-String -Path $_.FullName -Pattern "美股|US|usIndex|usDelta" }
```

结果：

- Python 编译通过。
- 前端脚本语法通过。
- 前端文件中未再检出 `美股`、`US`、`usIndex`、`usDelta`。

## 后续需要重新训练

这次修改影响 LightGBM 的训练预处理流程，需要重新训练模型验证：

```powershell
python -m python.ml.train_daily_predict
```

重点观察训练日志中是否还出现 LightGBM 特征维度 warning，并确认模型列表里是否重新包含 `lightgbm`。
