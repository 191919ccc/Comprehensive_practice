# 代码修改说明：机器学习参数与读取优化

## 1. 修改目标

本次修改主要针对机器学习训练配置中偏保守、重复特征和数据读取冗余的问题，目标是让日线三分类模型训练更充分、LSTM 表达能力更强、walk-forward 评估更接近真实训练状态，同时减少每次训练从数据库读取过多历史数据的开销。

## 2. 修改文件

- `python/ml/stock_ml.py`
- `python/common/config.py`
- `python/.env`
- `python/.env.example`

## 3. 具体修改内容

### 3.1 提高 LSTM 表达能力

将 LSTM 隐藏层从 `64` 提高到 `128`，并将输出 head 从 `32` 提高到 `64`。

修改后：

```python
LSTM_HIDDEN_SIZE = 128
LSTM_HEAD_SIZE = 64
```

这样可以减少输入特征维度较高时的瓶颈问题，适合当前较多技术指标和指数特征的输入规模。

### 3.2 单独配置 Early Stopping patience

原来代码使用 `ML_LSTM_FINE_TUNE_EPOCHS` 作为 early stopping patience，语义不清晰。本次新增：

```env
ML_LSTM_EARLY_STOP_PATIENCE=8
```

训练时使用：

```python
early_stop_patience = max(3, int(settings.ml_lstm_early_stop_patience))
```

这样 early stopping 的等待轮数可以独立调整，不再和 fine tune 配置混在一起。

### 3.3 删除重复特征 `price_vs_ma20`

原来 `price_vs_ma20` 与 `ma_20_gap` 完全相同：

```python
sorted_df["price_vs_ma20"] = sorted_df["ma_20_gap"]
```

本次删除该重复特征，并从 `SEQUENCE_FEATURES` 中移除，避免 LSTM 输入中出现完全重复维度。

### 3.4 优化 RandomForest 参数

原参数偏保守且只使用单核：

```python
RandomForestRegressor(n_estimators=40, n_jobs=1)
RandomForestClassifier(n_estimators=80, n_jobs=1)
```

修改为：

```python
RandomForestRegressor(n_estimators=100, n_jobs=-1)
RandomForestClassifier(n_estimators=200, n_jobs=-1)
```

这样可以提高随机森林稳定性，并使用本机多核 CPU 加快训练。

### 3.5 优化 LightGBM 参数

将 LightGBM 从较少轮数、高学习率改为更多轮数、低学习率：

```python
n_estimators=300
learning_rate=0.02
num_leaves=31
min_child_samples=20
n_jobs=-1
```

该配置通常比 `160 + 0.04` 更稳，适合当前 10 万级训练样本。

### 3.6 增加训练数据日期过滤

新增配置：

```env
ML_TRAINING_LOOKBACK_DAYS=730
```

`load_price_ticks()` 会按该配置只读取最近 730 天的数据。如果需要恢复全量历史训练，可以设置为：

```env
ML_TRAINING_LOOKBACK_DAYS=0
```

这样可以减少数据库 IO 和训练前的数据清洗压力。

### 3.7 提高 LSTM walk-forward 训练轮数

原来 walk-forward 中 LSTM 最多只训练 4 轮，容易低估 LSTM 的真实泛化能力。本次改为：

```python
walk_forward_epochs = max(2, min(15, settings.ml_lstm_epochs // 4))
```

在 `ML_LSTM_EPOCHS=60` 时，walk-forward 评估会训练 15 轮，更接近正式训练状态。

### 3.8 清理训练源优先级

当前项目主要保留国内股票数据，因此将训练源优先级中的 `yahoo` 移除：

```python
TRAINING_SOURCE_PRIORITY = ["akshare_cold_start", "sina", "tencent", "eastmoney"]
```

## 6. 注意事项

- `ML_TRAINING_LOOKBACK_DAYS=730` 会减少训练数据量，如果发现训练样本下降太多，可以改为 `1000` 或 `0`。
- LSTM hidden size 提高后，单轮训练耗时会增加，但 RTX 4060 Laptop GPU 可以承受。
- RandomForest 和 LightGBM 改为 `n_jobs=-1` 后会占用更多 CPU，训练时电脑风扇和 CPU 占用会上升。
