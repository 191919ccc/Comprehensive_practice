# LSTM 负提升集成降权说明

## 修改目标

最新训练结果显示，LSTM 在扩大序列样本并使用 60 日窗口后，验证集和 walk-forward 指标仍低于多数类基线：

- `lstm_direction_lift_over_baseline = -0.0564`
- `lstm_walk_forward_direction_lift_over_baseline = -0.0603`
- 但集成中 `lstm_weight` 仍有约 `0.2275`

这说明原来的动态集成权重对负提升模型惩罚不够，LSTM 仍可能拖累最终预测结果。

## 修改文件

- `python/ml/stock_ml.py`

## 具体修改

修改 `ensemble_weight_score()`：

1. 对 `walk_forward_direction_lift_over_baseline < 0` 的模型加强惩罚。
2. 对 LSTM 额外加入负提升惩罚：
   - `lift <= 0` 时进一步降权。
   - `lift <= -0.03` 时再次降权。
3. 将最低保底分从 `0.03` 降为 `0.005`，避免明显弱模型靠保底权重占比过高。

## 解决的问题

修改前，LSTM 即使 walk-forward 明显低于基线，仍能获得二十多个点的集成权重。

修改后，LSTM 如果没有超过时间滚动基线，只保留很低的辅助权重，主预测会更多依赖 LightGBM 和 Random Forest。
