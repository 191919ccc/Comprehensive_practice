# 代码修改说明：1.0% 阈值与指数特征实验

## 1. 修改时间

2026-05-09

## 2. 修改目标

本次修改按照“先试验，如果效果不好就恢复”的要求，对以下方向做实验：

- 尝试解决 AKShare / Eastmoney 代理问题，继续补充更多股票历史数据。
- 尝试单独拉取上证指数、创业板指、沪深 300 指数日线数据。
- 尝试把指数收益率作为外部特征加入训练。
- 让 LSTM 也支持 walk-forward 多轮评估。
- 用 1.0% 固定方向阈值训练一版模型，对比 `WATCH` 识别是否改善。

## 3. 修改文件

| 文件 | 修改内容 |
|---|---|
| `python/ml/cold_start.py` | 增加 `--include-index`、`--index-only`、`--use-proxy` 参数，并在默认情况下清理代理环境变量 |
| `python/ml/train_daily_predict.py` | 增加 `--direction-threshold`、`--no-write`、`--version-label` 参数，支持只实验不写库 |

## 4. 具体代码修改

### 4.1 cold_start 支持指数导入和禁用代理

新增参数：

```powershell
--include-index
--index-only
--use-proxy
```

默认行为：

- 清理 `HTTP_PROXY`、`HTTPS_PROXY`、`ALL_PROXY` 等代理环境变量。
- 设置 `NO_PROXY=*`。
- 希望绕过本机代理导致的 AKShare / Eastmoney 请求失败。

新增指数：

| 内部代码 | AKShare 代码 | 名称 |
|---|---|---|
| `INDEX_SH000001` | `sh000001` | 上证指数 |
| `INDEX_SZ399006` | `sz399006` | 创业板指 |
| `INDEX_SH000300` | `sh000300` | 沪深300 |

执行命令：

```powershell
D:\anaconda3\envs\MachineLearn\python.exe -m python.ml.cold_start --include-index --index-only --days 2500
```

结果：

```text
INDEX_SH000001 failed: RemoteDisconnected
INDEX_SZ399006 failed: RemoteDisconnected
INDEX_SH000300 failed: RemoteDisconnected
done inserted=0
```

结论：

禁用代理后指数接口仍然被远端断开，本次没有成功补入指数数据。问题更像是当前网络环境或接口访问限制，不是训练代码逻辑问题。

### 4.2 train_daily_predict 支持只实验不写库

新增参数：

```powershell
--direction-threshold 0.01
--no-write
--version-label daily-experiment
```

作用：

- `--direction-threshold 0.01`：使用固定 1.0% 阈值生成 `UP / DOWN / WATCH` 标签。
- `--no-write`：只训练、打印指标，不保存模型、不覆盖 MySQL 中的预测结果。
- `--version-label`：区分实验版本名。

这次实验没有覆盖线上预测表，当前 `ml_predictions` 仍然保持上一版：

```text
ensemble-daily-20260509091048
```

### 4.3 LSTM 增加 walk-forward 评估

修改位置：

```text
python/ml/stock_ml.py -> walk_forward_lstm_direction_metrics()
```

新增逻辑：

- 对 LSTM 方向分类分 3 段做 walk-forward 验证。
- 每一段用前面时间窗口训练，后面连续窗口验证。
- 为控制耗时，walk-forward 中只训练 LSTM 方向分类分支，不重复训练完整价格回归分支。

新增指标：

- `lstm_walk_forward_direction_accuracy`
- `lstm_walk_forward_balanced_direction_accuracy`
- `lstm_walk_forward_direction_macro_f1`
- `lstm_walk_forward_down_recall`
- `lstm_walk_forward_up_recall`
- `lstm_walk_forward_watch_recall`

## 5. 实验一：1.0% 固定方向阈值

执行命令：

```powershell
D:\anaconda3\envs\MachineLearn\python.exe -m python.ml.train_daily_predict --direction-threshold 0.01 --no-write --version-label daily-experiment
```

实验版本：

```text
daily-experiment-fixed0100bp-20260509091930
```

关键指标：

| 模型 | balanced accuracy | macro F1 | WATCH recall | walk-forward balanced accuracy |
|---|---:|---:|---:|---:|
| Random Forest | 0.3520 | 0.3341 | 0.2043 | 0.3643 |
| LightGBM | 0.3702 | 0.3299 | 0.7703 | 0.3670 |
| LSTM | 0.3515 | 0.3494 | 0.3768 | 0.3799 |

对比当前生产版本：

```text
当前生产版本：daily-20260509091048
当前生产 LSTM balanced_accuracy = 0.3960
```

结论：

- 1.0% 固定阈值确实显著改善了 LightGBM 对 `WATCH` 的识别，`watch_recall=0.7703`。
- 但整体最佳模型 balanced accuracy 只有 `0.3702`，低于当前生产版本 LSTM 的 `0.3960`。
- 因此没有推广 1.0% 阈值模型，也没有写入数据库。

## 6. 实验二：默认阈值 + 指数特征槽位

本次曾尝试把指数收益率作为外部特征加入 `SEQUENCE_FEATURES`：

- `index_sh_return`
- `index_cyb_return`
- `index_hs300_return`
- `index_avg_return`

但由于指数数据没有成功导入，这些特征实际全为 0。为了判断是否值得保留，执行了默认阈值 no-write 训练：

```powershell
D:\anaconda3\envs\MachineLearn\python.exe -m python.ml.train_daily_predict --no-write --version-label daily-default-experiment
```

实验版本：

```text
daily-default-experiment-20260509092114
```

关键指标：

| 模型 | balanced accuracy | macro F1 | walk-forward balanced accuracy |
|---|---:|---:|---:|
| Random Forest | 0.3524 | 0.3320 | 0.3764 |
| LightGBM | 0.3329 | 0.2598 | 0.3890 |
| LSTM | 0.3860 | 0.3475 | 0.3403 |

结论：

- LSTM balanced accuracy 从当前生产版本的 `0.3960` 降到 `0.3860`。
- 指数数据没有导入成功时，空的指数特征没有带来收益。
- 这类特征还会改变 LSTM 输入维度，导致旧模型加载存在兼容风险。

恢复动作：

- 已从 `SEQUENCE_FEATURES` 中撤回指数收益率特征。
- 已删除训练过程中的指数特征自动合并逻辑。
- 当前模型输入维度恢复到上一版兼容状态。
- MySQL 中预测结果没有被实验版本覆盖。

## 7. 当前保留的修改

本次保留以下不会破坏当前生产模型的能力：

- `cold_start.py` 的指数导入参数和代理清理参数。
- `train_daily_predict.py` 的固定阈值实验参数。
- `train_daily_predict.py` 的 `--no-write` 安全实验模式。
- LSTM walk-forward 评估函数。

这些改动方便后续继续实验，但不会自动覆盖当前线上预测。

## 9. 结论

本次实验说明：

- 1.0% 固定阈值可以提高 `WATCH` 识别，但会牺牲整体 balanced accuracy。
- 指数外部特征需要真实指数数据支撑；没有数据时不应加入模型输入。
- AKShare / Eastmoney 当前仍受网络或远端连接限制影响，代码层面已提供禁用代理入口，但数据源稳定性仍需单独解决。
- 当前生产版本 `daily-20260509091048` 暂时仍优于本次实验模型，因此已按要求保持现有预测结果不变。

下一步如果继续优化，应优先解决真实指数和更多股票历史数据的导入问题，而不是继续在当前 16 只股票数据上调阈值。
