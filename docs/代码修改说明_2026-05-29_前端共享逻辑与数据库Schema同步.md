# 前端共享逻辑与数据库 Schema 同步说明

## 修改目的

本次修复两个工程风险：

1. 前端展示逻辑重复，首页、子页面和旧入口脚本中各自维护 API 请求、模型指标口径、告警原因、历史记录和导出逻辑，容易出现同一问题只修一处的情况。
2. 数据库初始化脚本和运行时代码补字段不一致，新环境重建数据库时可能缺少 `alert_signal` 等当前代码已经依赖的字段。

## 修改文件

- `frontend/shared-ui.js`
- `frontend/index.html`
- `frontend/dash-pages.js`
- `frontend/alerts.html`
- `frontend/models.html`
- `frontend/market.html`
- `frontend/system.html`
- `python/sql/init.sql`
- `python/ml/stock_ml.py`
- `java-backend/src/main/java/com/recruitment/backend/service/DashboardService.java`

## 前端修改

新增 `frontend/shared-ui.js`，集中维护：

- `API_BASE`
- `requestJson()` / `postJson()`
- `metricFor()`
- `modelBenchmark()`
- `metricLabel()`
- `buildAlertReason()`
- `historyPath()`
- `historyExportUrl()`

首页逻辑 `dashboard-home.js`、子页面 `dash-pages.js` 现在优先调用 `window.StockShared` 中的公共逻辑。`alerts.html`、`models.html`、`market.html`、`system.html` 已在 `dash-pages.js` 前引入 `shared-ui.js`。

这样模型指标口径、告警原因和历史导出 URL 后续只需要改一处。

## 数据库 Schema 修改

`python/sql/init.sql` 已同步当前真实字段：

- `ml_predictions.alert_signal`
- `ml_prediction_history.alert_signal`
- `idx_ml_predictions_alert_signal_created_at`
- `idx_ml_prediction_history_alert_signal_time`

同时修正 `predicted_signal` 注释，明确它表示模型原始预测方向；`alert_signal` 表示告警过滤后的保守信号。

`stock_ml.py` 中的 `ml_prediction_history` 自动建表 SQL 也同步加入 `alert_signal` 字段，避免新库只跑训练脚本时缺字段。

`DashboardService.ensureExtendedTables()` 继续保留运行时兼容补字段逻辑，对旧库自动补：

- `ml_predictions.alert_signal`
- `ml_prediction_history.alert_signal`
- `alert_events.price_threshold`
- `alert_events.volume_threshold`
