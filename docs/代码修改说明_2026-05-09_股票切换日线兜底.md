# 股票切换日线兜底修改说明

## 修改目标

解决主仪表板切换到部分 A 股代码时弹出“未找到股票实时数据”的问题。

## 问题原因

前端主页面的股票下拉框数据来自多个来源，包括：

- `ml_predictions`
- `focus_stocks`
- `risk_stocks`
- `optimal_stocks`
- `latest_ticks`

其中很多 A 股代码来自日线训练结果和模型预测表，但 `/api/stocks/{symbol}` 这个接口主要返回实时行情明细。对于 `000100` 这类只有日线数据或预测结果、暂时没有实时明细的股票，接口会返回未找到。

因此页面实际不是“完全获取不到数据”，而是“切换股票时只认实时接口，不认已有的日线/预测数据”。

## 修改文件

- `frontend/index.html`

## 具体修改

### 1. 增加已知股票兜底查找

新增 `findKnownStock(data, symbol)`，从当前 dashboard 已加载的数据中查找股票。

数据来源包括预测结果、关注股票、风险股票、优选股票和最新行情。

### 2. 修改 `switchMainStock`

原逻辑：

- 请求 `/api/stocks/{symbol}`
- 如果接口返回空，直接弹窗“未找到实时数据”

新逻辑：

- 先请求 `/api/stocks/{symbol}` 获取实时明细
- 如果失败，从当前 dashboard 已加载的数据中找该股票
- 再请求 `/api/stocks/{symbol}/daily?days=120` 检查日线数据
- 只要存在实时数据、日线数据或预测表中的股票记录，就允许切换图表
- 只有实时和日线都不存在时，才提示“未找到股票实时或日线数据”

## 验证命令

```powershell
$html = Get-Content -Raw -Encoding UTF8 frontend\index.html
$m = [regex]::Match($html, '<script>([\s\S]*)</script>')
$tmp = Join-Path $env:TEMP ('index-inline-' + [guid]::NewGuid().ToString() + '.js')
Set-Content -LiteralPath $tmp -Encoding UTF8 -Value $m.Groups[1].Value
node --check $tmp
Remove-Item -LiteralPath $tmp
```

## 后续注意

- 如果希望“实时”和“日线”数据状态在 UI 上更清楚，可以在图表标题或状态标签中显示“实时可用 / 仅日线可用”。
- 当前修改不影响模型训练，也不需要重新训练。
