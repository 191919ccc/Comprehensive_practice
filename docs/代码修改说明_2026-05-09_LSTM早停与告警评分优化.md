# 代码修改说明：LSTM 早停、分方向告警阈值与模型评分优化

## 1. 修改目标

本次修改针对上一轮训练暴露的问题：

- `ML_LSTM_EPOCHS=60` 后，LSTM 分类稳定性略有改善，但 `return_mae` 变差，说明存在过训练风险。
- DOWN 告警平均收益为正，说明部分 DOWN 信号实际是错边信号。
- 模型选择逻辑原来更偏向分类指标，没有把 UP / DOWN 告警后的真实收益质量纳入评分。

## 2. 修改文件

- `python/.env`
- `python/common/config.py`
- `python/ml/stock_ml.py`

## 3. 具体修改

### 3.1 新增 UP / DOWN 分方向告警置信度阈值

`python/.env` 新增：

```env
ML_ALERT_UP_CONFIDENCE_THRESHOLD=0.60
ML_ALERT_DOWN_CONFIDENCE_THRESHOLD=0.75
```

`python/common/config.py` 新增对应读取：

- `settings.ml_alert_up_confidence_threshold`
- `settings.ml_alert_down_confidence_threshold`

保留旧的 `ML_ALERT_CONFIDENCE_THRESHOLD=0.60` 作为兼容配置。

### 3.2 告警过滤改为分方向过滤

`python/ml/stock_ml.py` 中的 `apply_alert_confidence_filter()` 已改为：

- `UP` 且 `confidence < 0.60`，降级为 `WATCH`
- `DOWN` 且 `confidence < 0.75`，降级为 `WATCH`

这样 DOWN 告警会更严格，减少“预测跌但实际涨”的错边风险。

### 3.3 告警回测指标同步使用分方向阈值

`alert_backtest_metrics()` 已同步使用 UP / DOWN 独立阈值，并新增输出：

- `alert_up_confidence_threshold`
- `alert_down_confidence_threshold`

因此训练日志里的 `alert_backtest_*` 指标会反映真实告警过滤后的效果。

### 3.4 LSTM 增加 Early Stopping

`train_lstm()` 中新增 early stopping。

监控指标为综合验证损失：

```python
val_combined_loss = direction_val_loss + 0.3 * price_val_loss
```

这样不会只看方向分类，也会约束收益回归退化。

实现细节：

- 先从 LSTM 训练集内部再切出验证集。
- early stopping 只看内部验证集，不使用最终测试集，避免最终指标偏乐观。
- `ML_LSTM_FINE_TUNE_EPOCHS=10` 现在作为 early stopping patience 使用。
- 保存 `val_combined_loss` 最低时的 price / direction 两个模型权重。
- 训练结束后恢复最佳 epoch 的模型权重。

训练输出会新增：

- `lstm_best_epoch`
- `lstm_early_stop_epoch`
- `lstm_best_val_combined_loss`

### 3.5 模型选择评分加入告警收益质量

`direction_quality_score()` 已调整。

现在优先使用 walk-forward 指标：

- `walk_forward_balanced_direction_accuracy`
- `walk_forward_direction_macro_f1`
- `walk_forward_direction_lift_over_baseline`

同时新增：

- DOWN 告警惩罚：如果 `alert_backtest_down_avg_inverse_return` 为负，说明 DOWN 告警平均错边，会拉低模型得分。
- UP 告警奖励：如果 `alert_backtest_up_avg_return` 为正，会给少量加分。

这样模型选择不只看分类准确率，也会考虑告警是否真的有业务价值。

### 3.6 新增大盘 3 日趋势特征

`SEQUENCE_FEATURES` 新增：

```python
index_up_3d
```

含义：

```text
当前股票对应市场指数近 3 日收益合计是否大于 0
```

实现上使用 `market_index_return_1` 按股票时间序列滚动 3 日计算。这个特征用于帮助模型识别“大盘上涨环境下 DOWN 信号可信度下降”的情况。

## 6. 判断标准

如果修改有效，理想现象是：

- `lstm_return_mae` 不再明显恶化。
- `lstm_early_stop_epoch` 小于 `60`，说明早停生效。
- `alert_backtest_down_avg_inverse_return` 不再为负。
- DOWN 告警覆盖率会下降，但命中质量应该提高。
- 最终选中的模型不一定永远是 LSTM，而是更偏向告警质量更稳的模型。
