# 前端可视化可读性增强修改说明

## 修改目标

本次修改围绕主仪表板的可读性和答辩展示稳定性展开，重点解决“实时”和“日线+预测”模式不够清晰、实时无数据时容易误解为预测图、首页告警缺少快速筛选的问题。

## 修改文件

- `frontend/index.html`

## 具体修改

1. 在股票走势图工具栏中增加 `chartModeHint`，明确当前使用的是“分钟线接口”还是“日线接口”。
2. 在图表区域增加 `chartState` 空状态提示层，用于说明当前图表为什么没有实时数据或为什么只显示预测。
3. 新增 `renderEmptyRealtimeChart()`，当实时模式没有分钟级行情时，不再用 AI 预测线伪装成实时图，而是显示“暂无分钟级实时数据，可切换到日线+预测”的提示。
4. 在日线模式和预测兜底模式中同步更新模式说明，让用户能看出数据来自日线历史、分钟线聚合日线或仅模型预测。
5. 在首页风险告警中心增加快速筛选标签：全部、高危、中危、低危、价格、成交量、模型。
6. 首页告警筛选支持前端即时切换，不需要重新请求后端接口。

## 验证情况

- 已执行 `node --check frontend/dash-pages.js`。
- 已执行内联脚本语法检查，结果为 `index inline script ok`。
- 已检查 `chartState`、`homeAlertTabs`、`renderEmptyRealtimeChart`、`homeAlertFilter` 等关键标记已写入首页。

## 剩余说明

本次只修改前端展示逻辑，不涉及后端接口和模型训练。若要查看真实数据效果，需要保持 Java 后端和 MySQL 数据库正常运行，然后刷新 `frontend/index.html`。
