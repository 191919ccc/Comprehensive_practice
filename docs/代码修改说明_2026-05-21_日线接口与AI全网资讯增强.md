# 日线接口与 AI 全网资讯增强修改说明

日期：2026-05-21

## 修改目标

1. 修复前端请求 `/api/stocks/{symbol}/daily?days=120` 时后端没有对应接口，导致日线图拿不到数据的问题。
2. 修复 AI 联网检索只使用单站 RSS 或只返回链接摘要的问题。
3. 让 `application.yml` 中的 `app.ai.serper-key` 真正生效，用于 Serper 全网检索。

## 修改文件

- `java-backend/src/main/java/com/recruitment/backend/controller/DashboardController.java`
- `java-backend/src/main/java/com/recruitment/backend/service/DashboardService.java`
- `java-backend/src/main/java/com/recruitment/backend/service/AiChatService.java`

## 具体修改

### 1. 恢复日线接口

新增接口：

```http
GET /api/stocks/{symbol}/daily?days=120
```

后端逻辑：

- 优先查询 `daily_stock_bars`。
- 如果 `daily_stock_bars` 没有该股票数据，则从 `price_ticks` 按交易日期聚合兜底。
- 返回 `daily_source`，用于前端区分数据来自 `daily_stock_bars` 还是 `price_ticks_aggregated`。
- 同时返回该股票最新一条 `ml_predictions` 预测结果，供前端画 T+3 预测点。

### 2. 修复分钟趋势开盘价字段

`fetchStockTrend()` 重新返回 `open_price`，避免实时趋势图 tooltip 缺少开盘价。

### 3. AI 使用配置文件 Serper Key

原代码只读取：

```java
System.getenv("SERPER_API_KEY")
```

这会导致 `application.yml` 中的：

```yaml
app.ai.serper-key
```

实际不生效。

现在已改为通过 Spring `@Value` 注入：

```java
@Value("${app.ai.serper-key:${SERPER_API_KEY:}}")
```

优先使用配置文件，环境变量作为兜底。

### 4. 全网检索不再只抓单一 RSS

当 Serper Key 存在时：

- 先请求 `https://google.serper.dev/news`
- 不足数量时继续请求 `https://google.serper.dev/search`
- 合并新闻结果和网页结果
- 去重标题和链接
- 每条资讯保留：
  - `title`
  - `link`
  - `source`
  - `published_at`
  - `summary_content`

当 Serper 不可用时，才降级到 RSS。

### 5. 强化 System Prompt

提示词新增约束：

- 如果 `external_news.status=ok`，必须阅读 `summary_content`。
- 不允许只罗列链接。
- 必须做跨来源归纳。
- 引用新闻时用 Markdown 链接自然嵌入分析句子。
- 必须说明新闻对股票的基本面、资金面、情绪面和风险点的影响路径。
