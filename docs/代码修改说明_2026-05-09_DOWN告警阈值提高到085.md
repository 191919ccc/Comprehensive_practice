# 代码修改说明：DOWN 告警置信度阈值提高到 0.85

## 1. 修改目标

上一轮训练结果显示，LSTM 的 UP 告警质量较好，但 DOWN 告警仍然存在平均错边问题：

```text
lstm_alert_backtest_down_avg_inverse_return = -0.0034
```

这说明部分 DOWN 告警在真实行情里反而上涨。为了减少低质量 DOWN 告警，本次只提高 DOWN 告警的后处理置信度阈值。

## 2. 修改文件

- `python/.env`

## 3. 具体修改

将：

```env
ML_ALERT_DOWN_CONFIDENCE_THRESHOLD=0.75
```

修改为：

```env
ML_ALERT_DOWN_CONFIDENCE_THRESHOLD=0.85
```

UP 阈值保持不变：

```env
ML_ALERT_UP_CONFIDENCE_THRESHOLD=0.60
```

## 4. 影响范围

该修改不会改变模型训练标签，也不会改变模型结构。

影响的是最终告警过滤逻辑：

- `UP`：`confidence < 0.60` 时降级为 `WATCH`
- `DOWN`：`confidence < 0.85` 时降级为 `WATCH`

因此 DOWN 告警数量会进一步减少，但留下来的 DOWN 告警应当更严格。

## 5. 后续命令

如果只想用已有模型重新生成预测：

```powershell
python -m python.ml.predict_daily
```

如果要重新训练并重新评估指标：

```powershell
python -m python.ml.train_daily_predict
```

## 6. 观察指标

重新训练后重点看：

- `alert_backtest_down_count`
- `alert_backtest_down_hit_rate`
- `alert_backtest_down_avg_inverse_return`
- `alert_backtest_coverage`

理想情况是 DOWN 数量下降，但 `down_avg_inverse_return` 不再为负。
