# 代码修改说明：提高机器学习训练样本上限

## 1. 修改目标

上一轮补齐 92 只股票后，数据库里已经有 `151239` 条 `akshare_cold_start` 日线数据，但训练日志显示实际训练被限制在约 `23920` 条样本，LSTM 也只使用约 `5980` 条序列。

这说明数据已经补上来了，但模型没有充分使用新增样本。本次修改的目标是提高训练样本上限，让模型真正利用更多历史日线数据重新训练。

## 2. 修改文件

修改文件：

- `python/.env`

新增文档：

- `docs/代码修改说明_2026-05-09_提高训练样本上限.md`

## 3. 具体修改

原配置：

```text
ML_MAX_TRAIN_ROWS=24000
ML_LSTM_MAX_SEQUENCES=6000
```

修改后配置：

```text
ML_MAX_TRAIN_ROWS=60000
ML_LSTM_MAX_SEQUENCES=12000
```

含义：

- `ML_MAX_TRAIN_ROWS` 控制传统模型和整体训练集的最大样本行数。
- `ML_LSTM_MAX_SEQUENCES` 控制 LSTM 最多使用多少条时间序列窗口。

本次曾尝试将配置提高到 `80000 / 20000`，但训练耗时超过 30 分钟且没有成功落库。因此最终采用更稳妥的 `60000 / 12000`，保证训练可以完成并写入数据库。

## 6. 结果分析

本次修改后，模型确实使用了更多数据：

```text
23920 -> 59984
5980 -> 11960
```

指标变化可以这样理解：

1. LSTM 主验证集 balanced accuracy 从 `0.3809` 提升到 `0.4043`，说明增加样本对 LSTM 有正向作用。
2. LSTM walk-forward balanced accuracy 从 `0.3654` 提升到 `0.3724`，稳定性也略有改善。
3. LightGBM 主验证集下降到 `0.3434`，但 walk-forward 提升到 `0.3853`，说明它在多时间窗口上更稳，但单次切分表现变差。
4. RandomForest 仍然偏向 WATCH，提升有限，不应作为主要参考模型。

综合来看，本次配置调整是有效的：它没有让模型发生质变，但让 LSTM 和 LightGBM 的稳定性指标都有改善。

## 7. 当前结论

当前最值得关注的模型不是 RandomForest，而是：

- LSTM：主验证集最好，balanced accuracy 为 `0.4043`；
- LightGBM：walk-forward 最好，balanced accuracy 为 `0.3853`。

代码当前选择的最佳非基线模型是：

```text
lstm
```

前端生产预测仍使用：

```text
ensemble
```

## 8. 后续建议

下一步不建议继续单纯加大训练样本上限，因为 `80000 / 20000` 已经明显过慢。更值得做的是：

1. 调整模型选择逻辑，把 walk-forward 指标权重加大；
2. 对 LightGBM 和 LSTM 做单独阈值实验；
3. 给训练增加早停机制，避免 LSTM 在更大样本下训练过久；
4. 增加长期指数特征和行业指数特征；
5. 将 `drift_check` 的最近预测样本积累满以后再判断漂移。
