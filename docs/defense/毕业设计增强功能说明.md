# 股票实时流数据分析平台增强功能说明

## 1. 当前项目阶段

本项目已经从“课程演示型实时大屏”推进到“毕设原型系统”阶段。当前主链路已经具备：

- 多源股票行情采集：从 Yahoo、EastMoney、Sina、Tencent、Stooq 等数据源采集行情。
- Kafka 实时传输：行情事件写入 `stock_realtime_topic`。
- Spark Structured Streaming 实时计算：消费 Kafka 数据，完成清洗、窗口统计、异常识别和 HDFS 归档。
- MySQL 结果落库：保存实时行情、聚合指标、异常告警、模型预测结果。
- Java 后端接口：向前端提供仪表盘、股票查询、告警处理、历史查询、模型指标等 API。
- ECharts 前端大屏：展示实时状态、市场概览、重点关注股票、异常告警、模型预测和历史数据。

因此当前项目已经不是单纯的数据展示页面，而是包含“采集、传输、实时计算、存储、后端服务、前端展示、机器学习分析”的完整实时数据分析系统。

## 2. 新增增强功能

### 2.1 最优股与高风险股排序

系统新增了股票综合评分功能，用来解决“前端数据看了不知道有什么用”的问题。

最优股排序关注“哪些股票更值得进一步观察”，综合考虑：

- 涨跌幅：反映短期价格变化强度。
- 成交量：反映市场关注度和交易活跃度。
- 异常告警次数：反映风险事件频率。
- 机器学习预测信号：反映模型对下一时刻价格方向的判断。

风险股排序关注“哪些股票短期波动风险更高”，综合考虑：

- 涨跌幅绝对值。
- 是否触发价格波动告警。
- 是否触发成交量异常告警。
- 机器学习预测是否为下跌信号。

对应后端接口：

```text
GET /api/stocks/ranking?type=optimal&limit=10
GET /api/stocks/ranking?type=risk&limit=10
```

前端展示位置：

- “综合评分优选股”
- “高风险关注股”

说明：该评分不是投资建议，而是数据分析系统中的辅助决策指标，用于体现实时分析价值。

### 2.2 模型对比实验

项目已经加入多模型预测能力，当前包含：

- Random Forest：随机森林模型，适合处理非线性特征关系，训练速度较快，可解释性较强。
- LightGBM：梯度提升树模型，适合结构化表格数据，通常在金融特征预测中表现较好。
- LSTM：循环神经网络模型，适合处理时间序列数据，可学习历史价格变化趋势。

训练程序会记录不同模型的误差指标，并将结果保存到 MySQL 的模型指标表中。前端通过模型指标面板展示不同模型的效果。

对应后端接口：

```text
GET /api/ml/models
```

对应运行命令：

```powershell
D:\anaconda3\envs\MachineLearn\python.exe -m python.ml.train_predict
```

### 2.3 告警处理闭环

原来的告警只展示异常信息，现在新增告警处理状态，形成闭环：

- `OPEN`：新告警，尚未处理。
- `ACKED`：已确认，表示用户已经看到。
- `IGNORED`：已忽略，表示该告警暂不处理。
- `RESOLVED`：已解决，表示该异常已经完成处理。

对应后端接口：

```text
GET /api/alerts?status=OPEN&limit=20
POST /api/alerts/{id}/status
```

前端告警列表支持：

- 确认告警
- 忽略告警
- 标记解决

这样答辩时可以说明系统不只是“发现异常”，还具备“告警处理与状态跟踪”能力。

### 2.4 历史趋势查询与导出

系统新增历史数据查询和 CSV 导出能力，用于支撑后续分析和实验记录。

对应后端接口：

```text
GET /api/history?symbol=AAPL&minutes=1440&limit=200
GET /api/history/export?symbol=AAPL&minutes=1440&limit=1000
```

前端提供：

- 历史查询按钮
- CSV 导出按钮

该功能可以用于：

- 查看单只股票最近一段时间价格变化。
- 导出训练样本或实验数据。
- 支撑论文中“数据准备”和“实验分析”章节。

## 3. 数据的使用方式

### 3.1 实时数据处理

采集程序持续获取股票行情，将每条行情封装为事件写入 Kafka。Spark Structured Streaming 作为实时计算层，从 Kafka 消费数据后进行清洗、转换、窗口统计和异常识别。

实时处理结果主要用于：

- 判断当前行情流是否正常流动。
- 统计市场整体涨跌情况。
- 分析不同行业、不同市场的热点变化。
- 识别价格剧烈波动和成交量异常。
- 将结果写入 MySQL 供前端查询。
- 将流式结果归档到 HDFS，支持容错恢复和后续离线分析。

### 3.2 机器学习分析

机器学习模块使用 MySQL 中的历史行情数据构建训练样本，提取价格、涨跌幅、成交量、成交额等特征，训练模型预测下一时刻价格或涨跌方向。

模型结果主要用于：

- 给股票打上 `UP`、`DOWN`、`WATCH` 等预测信号。
- 与实时异常规则结合，形成综合关注评分。
- 对比 Random Forest、LightGBM、LSTM 的预测误差。
- 在前端展示模型预测结果，增强系统分析深度。

### 3.3 前端辅助决策

前端不再只展示原始行情，而是围绕“用户看了能做什么”进行组织：

- 看实时状态：判断采集和流处理链路是否正常。
- 看市场概览：判断当前整体偏强还是偏弱。
- 看优选股：快速找到值得关注的股票。
- 看风险股：快速发现波动过大或异常较多的股票。
- 看告警列表：定位具体异常事件。
- 看模型对比：说明机器学习模块的实验结果。
- 查历史数据：支持复盘和导出。

## 4. 答辩时可以强调的项目价值

本项目的价值不在于“预测股票一定准确”，而在于构建了一套完整的实时流分析系统。它能够从多源数据采集开始，经过 Kafka 和 Spark 进行实时处理，再将分析结果落库并可视化展示，最终形成异常监控、辅助决策和历史分析闭环。

相比普通数据大屏，本项目增加了：

- 实时数据链路。
- 多源数据接入。
- 流式异常检测。
- HDFS 归档与 checkpoint 容错。
- Java 后端接口服务。
- 机器学习预测与模型对比。
- 告警处理状态闭环。
- 历史查询与导出能力。

这些内容可以支撑毕业设计中的需求分析、系统设计、核心功能实现、实验测试和结果展示。

## 5. 后续还可以继续完善的方向

如果要进一步提升到更完整的毕设系统，可以继续添加：

- 用户登录与角色权限：管理员、普通用户、分析员。
- 自定义阈值配置：用户可以配置涨跌幅阈值、成交量阈值、关注股票池。
- 告警统计报表：按天、按股票、按行业统计告警数量。
- 模型训练记录表：保存每次训练数据范围、模型参数、训练时间和评估指标。
- 股票池管理：支持用户添加自选股、重点关注股。
- 数据质量监控：统计各数据源成功率、失败率、延迟情况。
- 定时训练任务：定期重新训练模型并更新预测结果。
