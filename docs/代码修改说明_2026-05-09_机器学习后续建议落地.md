# 代码修改说明：机器学习后续建议落地

## 1. 修改时间

2026-05-09

## 2. 修改目标

本次修改根据上一版文档中的后续建议继续落地，重点处理以下问题：

- 增加市场宽度类特征，让模型不仅看单只股票自身技术指标。
- 增加 `UP / DOWN / WATCH` 阈值实验指标，观察不同阈值下标签分布。
- 增加 walk-forward 回测，减少只看一次 8:2 时间切分带来的偶然性。
- 将 drift 检测从“预测后一条真实行情”改成“预测 horizon 后的真实行情”，与 `PREDICTION_HORIZON=3` 对齐。
- 尝试扩充更长周期日线数据。

## 4. 具体代码修改

### 4.1 增加市场宽度特征

修改位置：

```text
python/ml/stock_ml.py -> SEQUENCE_FEATURES
python/ml/stock_ml.py -> add_technical_features()
```

新增特征：

| 特征 | 含义 |
|---|---|
| `market_relative_change` | 个股涨跌幅相对同市场平均涨跌幅的偏离 |
| `market_up_ratio` | 同市场同一时间上涨股票比例，减去 0.5 后归一到偏多/偏空 |
| `sector_up_ratio` | 同行业同一时间上涨股票比例，减去 0.5 后归一到偏多/偏空 |

作用：

- 让模型知道个股走势是“自己强”还是“跟随整个市场强”。
- 增加市场情绪和行业联动信息。
- 为后续接入真实指数特征打基础。

同时修复了一个字段遗漏：

```text
load_price_ticks() 原来没有 SELECT market
```

新增市场特征后必须读取 `market` 字段，否则训练会报 `KeyError: 'market'`。本次已修复。

### 4.2 增加阈值实验指标

修改位置：

```text
python/ml/stock_ml.py -> threshold_experiment_metrics()
python/ml/train_daily_predict.py
python/ml/train_predict.py
```

新增固定阈值实验：

```text

0.5%
1.0%
```

每次训练会额外写入：

- `0030bp_down_ratio`
- `0030bp_up_ratio`
- `0030bp_watch_ratio`
- `0050bp_down_ratio`
- `0050bp_up_ratio`
- `0050bp_watch_ratio`
- `0100bp_down_ratio`
- `0100bp_up_ratio`
- `0100bp_watch_ratio`

作用：

- 观察不同阈值下 `UP / DOWN / WATCH` 样本比例如何变化。
- 判断当前标签是不是过度偏向某一类。
- 为后续选择更合理的方向阈值提供依据。

### 4.3 增加 walk-forward 回测

修改位置：

```text
python/ml/stock_ml.py -> walk_forward_direction_metrics()
python/ml/stock_ml.py -> train_tabular_models()
```

原来主要使用每只股票 8:2 时间切分。现在对表格模型额外增加 3 段 walk-forward 回测：

```text
前 50% 训练 -> 后 10% 验证
前 60% 训练 -> 后 10% 验证
前 70% 训练 -> 后 10% 验证
```

新增指标示例：

- `walk_forward_direction_accuracy`
- `walk_forward_balanced_direction_accuracy`
- `walk_forward_direction_macro_f1`
- `walk_forward_down_recall`
- `walk_forward_up_recall`
- `walk_forward_watch_recall`

说明：

- 本次先对 Random Forest 和 LightGBM 增加 walk-forward。
- LSTM 的 walk-forward 训练成本更高，暂未加入多轮重训。

### 4.4 drift 检测按 horizon 对齐

修改位置：

```text
python/ml/model_drift.py
```

原逻辑：

```text
预测生成后，取该股票下一条入库行情作为真实结果。
```

问题：

- 训练目标是 `PREDICTION_HORIZON=3`。
- drift 却只看下一条行情，评估口径不一致。

新逻辑：

1. 根据预测记录中的 `symbol`、`current_price`、`predicted_at` 找到预测时对应的基准行情时间。
2. 从该基准行情时间开始，向后取第 `horizon` 条行情。
3. 默认 `horizon=3`，与训练目标对齐。
4. 命令行新增参数：

```powershell
--horizon
```

drift 返回结果现在会包含：

```text
horizon: 3
signal_threshold: 0.005
```

### 6.2 walk-forward 指标

Random Forest：

```text

walk_forward_balanced_accuracy=0.3641
walk_forward_macro_f1=0.3442
```

LightGBM：

```text

```

解释：

- LightGBM 在单次 8:2 切分上较差，但 walk-forward balanced accuracy 达到约 0.3890。
- Random Forest 原始准确率更高，但仍偏向 `WATCH`。
- LSTM 当前单次验证 balanced accuracy 约 0.3960，仍是本次最佳方向模型。

### 6.3 阈值实验结果

| 阈值 | DOWN 占比 | UP 占比 | WATCH 占比 | 多数类基线 |
|---|---:|---:|---:|---:|
| 0.3% | 0.4694 | 0.4388 | 0.0918 | 0.4694 |
| 0.5% | 0.4384 | 0.4088 | 0.1528 | 0.4384 |
| 1.0% | 0.3652 | 0.3412 | 0.2936 | 0.3652 |

结论：

- 当前固定阈值越高，`WATCH` 占比越高。
- 0.5% 阈值下 `WATCH` 并不夸张，类别分布相对还能接受。
- 如果后续想让模型更重视“观望”，可以考虑 1.0% 阈值，但会减少有效 UP/DOWN 训练样本。

### 6.4 drift 检测结果

```text
drift_check={
  'ready': True,
  'evaluated': 50,
  'accuracy': 0.38,
  'threshold': 0.55,
  'signal_threshold': 0.005,
  'horizon': 3,
  'drift': True

```

说明：

- drift 已经按 horizon=3 对齐训练目标。
- 准确率仍然低，说明近期预测仍不稳定。
- 这不是 drift 代码口径不一致导致的假低分，而是真实预测质量仍需继续提升。

## 7. 本次修改后的结论

本次修改后，系统已经具备更完整的机器学习评估链路：

- 有市场宽度特征。
- 有阈值实验。
- 有 walk-forward 回测。
- drift 检测与训练 horizon 对齐。
- 训练结果能更细地解释模型为什么表现一般。

但是模型效果没有出现大幅提升，原因也更加明确：

- 当前股票数量仍只有 16 只。
- 扩展数据受外部接口和代理影响。
- 技术指标和市场宽度特征仍不足以稳定预测短期方向。
- LSTM 当前相对最好，但 balanced accuracy 仍只有约 0.396。

## 8. 后续建议

下一步不建议继续盲目调参，应该优先处理数据源：

1. 解决 AKShare / Eastmoney 代理问题，批量补齐更多股票。
2. 单独拉取真实指数数据，例如上证指数、创业板指、沪深 300。
3. 将指数收益率、行业指数收益率作为外部特征写入训练表。
4. 让 LSTM 也支持 walk-forward 多轮评估。
5. 尝试用 `1.0%` 固定方向阈值训练一版模型，对比 `WATCH` 识别是否改善。
