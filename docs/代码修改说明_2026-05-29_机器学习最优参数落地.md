# 代码修改说明：机器学习最优参数落地

## 修改目的

把 `reports/ml_tuning/daily_param_tuning_best_20260529084541.md` 中的最优参数写入正式训练代码。RandomForest 使用 `rf_leaf_stable + 0.02`，LightGBM 使用 `lgbm_production + 0.015`，LSTM 使用 `lstm_longer_context + 0.015`。由于三个模型的最优方向阈值不完全一致，日线训练入口改为按模型分别构造标签数据。

## 修改文件

- `python/ml/stock_ml.py`
- `python/ml/train_daily_predict.py`
- `python/common/config.py`

## 具体修改

1. `stock_ml.py` 新增 `OPTIMAL_MODEL_CONFIGS`，集中记录三类模型的最优参数和各自 `direction_threshold`。
2. `prepare_dataset()` 新增 `direction_threshold` 参数，兼容旧调用，同时支持外部指定标签阈值。
3. 新增 `prepare_model_datasets()`，按模型最优阈值分别生成训练数据：
   - RandomForest: `0.02`
   - LightGBM: `0.015`
   - LSTM: `0.015`
4. `train_random_forest()` 改为 `rf_leaf_stable` 参数：
   - `n_estimators_reg=180`
   - `n_estimators_cls=300`
   - `max_depth=18`
   - `min_samples_leaf=8`
   - `max_features=0.55`
   - `calibration=sigmoid`
   - `n_jobs=1`，避免 Windows 本地训练触发 `[WinError 5] 拒绝访问`
5. `train_tabular_models()` 和 `train_lstm()` 在 `metrics` 和 `extra` 中写入各自的 `direction_threshold`，便于后续指标展示、告警回测和预测过滤复用。
6. `train_models()` 支持传入 `{model_name: dataset}`，正式日线训练时各模型使用自己的标签阈值。
7. `apply_alert_confidence_filter()` 支持传入模型阈值；单模型预测使用自己的阈值，ensemble 使用参与模型阈值的加权平均。
8. `train_daily_predict.py` 默认使用按模型最优阈值；只有显式传 `--direction-threshold` 时才覆盖所有模型阈值。
9. `config.py` 更新 LSTM 默认参数为本次最优结构：
   - `ML_LSTM_WINDOW_SIZE=45`
   - `ML_LSTM_LEARNING_RATE=0.00045`
   - `ML_LSTM_WEIGHT_DECAY=0.002`

## 剩余限制

- 当前落地的是本次中等规模搜索得到的最优参数，不代表未来新增数据后永远最优。
- 如果环境变量显式设置了 `ML_LSTM_WINDOW_SIZE`、`ML_LSTM_LEARNING_RATE` 或 `ML_LSTM_WEIGHT_DECAY`，会覆盖代码默认值。
- `guarded_used=1.0` 仍表示该验证窗口使用过基线兜底，答辩时应表述为“最终决策不低于基线”，不要说原始模型信号全部显著超过基线。
