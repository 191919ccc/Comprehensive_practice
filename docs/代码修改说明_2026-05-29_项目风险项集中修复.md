# 项目风险项集中修复说明

## 修改目的

本次修改针对项目审查中发现的几个会影响答辩稳定性和可信度的问题：

- 子页面 AI Markdown 渲染缺少 HTML 清洗，存在 XSS 风险。
- 模型训练可能继续使用过期日线数据，前端也没有提示数据新鲜度。
- 告警短时间重复、高危比例过高，容易削弱告警可信度。
- 首页内联脚本过大，前端入口维护成本高。
- 前端图表库依赖公网 CDN，断网答辩时可能空白。
- 启动脚本后端 jar 固定路径缺少校验，存在启动错误产物的风险。

## 具体修改

1. AI Markdown 安全渲染
   - 在 `shared-ui.js` 新增 `sanitizeHtml()` 和 `renderMarkdown()`。
   - 子页面 `dash-pages.js` 的 AI 回复不再直接 `marked.parse()` 后写入 `innerHTML`，而是统一走清洗后的 Markdown 渲染。
   - 首页 `dashboard-home.js` 也优先复用 `shared-ui.js` 的 Markdown 渲染逻辑。

2. 模型数据时效校验
   - `train_daily_predict.py` 在训练前检查 `daily_stock_bars` 和 `daily_index_bars` 最新日期。
   - 默认超过 `ML_MAX_DAILY_DATA_AGE_DAYS=10` 天会停止训练。
   - 调试可用 `--allow-stale-data` 或 `ML_ALLOW_STALE_DAILY_DATA=true` 临时绕过。
   - 后端 `/api/dashboard` 增加 `daily_data_freshness`，前端模型卡片和模型页会显示“数据过期”提示。

3. 告警降噪
   - Spark 告警写入前增加同股票同规则冷却窗口，默认 `ALERT_DUPLICATE_COOLDOWN_MINUTES=10`。
   - 高危判断收紧：价格高危、价格和成交量同时触发、或有历史成交量基线支持的成交量高危才标为 `HIGH`。
   - 减少活跃股票因成交量大而持续刷高危告警。

4. 前端入口整理
   - 首页大屏脚本从 `index.html` 拆出为 `dashboard-home.js`。
   - `index.html` 保留结构、样式和脚本引用。
   - `frontend/README.md` 说明当前真实运行入口，明确 `app.js` 是早期脚本，当前页面不再引用。

5. 本地前端依赖
   - 新增 `vendor-loader.js`，页面优先加载 `frontend/vendor/` 下的本地库，缺失时才回退 CDN。
   - 已下载 `chart.umd.js`、`echarts.min.js`、`marked.min.js` 到 `frontend/vendor/`。
   - 移除首页 Google Fonts 网络依赖，改用本地系统字体栈。

6. 启动 jar 校验
   - `pom.xml` 明确 `finalName=stock-risk-backend-${project.version}`。
   - `start_demo.ps1` 在启动后端前校验目标 jar 是否存在且大小合理，避免启动错误或残留小 jar。

## 剩余限制

- 本次没有自动重新训练模型。由于训练入口新增了数据时效强校验，如果日线数据过期，需要先重新拉取 `daily_stock_bars` 和 `daily_index_bars`。
- CDN 仍作为兜底保留；答辩时应确认 `frontend/vendor/` 三个本地文件存在。
- 早期脚本 `frontend/app.js` 和旧样式 `frontend/style.css` 已在后续清理中删除，当前入口以 `dashboard-home.js`、`dash-pages.js` 和 `shared-ui.js` 为准。
