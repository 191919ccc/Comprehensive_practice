# GitHub 上传准备与配置脱敏说明

## 修改目的

本次修改用于把项目整理成可以上传到 GitHub 的状态，重点避免把本机数据库密码、第三方 API Key、模型产物、编译产物和运行日志提交到远程仓库。

## 修改文件

- `.gitignore`
- `java-backend/src/main/resources/application.yml`
- `python/.env.example`
- `docs/代码修改说明_2026-05-30_GitHub上传准备与配置脱敏.md`

## 具体修改

1. `.gitignore` 已覆盖 Python 缓存、虚拟环境、本机 `.env`、训练模型目录、Java `target`、日志、运行时 PID、IDE 配置和系统缓存文件。
2. Java 后端数据库连接改为从环境变量读取：
   - `MYSQL_JDBC_URL`
   - `MYSQL_USER`
   - `MYSQL_PASSWORD`
3. Serper 搜索 Key 改为从 `SERPER_API_KEY` 读取，默认留空，避免明文密钥进入 Git 历史。
4. `python/.env.example` 中的示例密码改为 `change-me`，真实配置应放在未提交的 `python/.env` 中。

## 剩余限制

- 当前机器没有安装或识别 GitHub CLI `gh`，无法由命令行自动创建 GitHub 远程仓库。
- 如果使用 HTTPS 远程推送，需要先在 GitHub 创建仓库，或安装 `gh` 后执行 `gh auth login`。
